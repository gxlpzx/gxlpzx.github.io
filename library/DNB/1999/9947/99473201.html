《 Netscape常见错误及解决办法 》

　　在用Netscape浏览因特网时经常会遇到一些连接错误信息，对于一个老“网虫”，可能有自己的应对之策，可对于一些初学者来说，面对这些错误信息往往是感到束手无策，不知道为什么会出现这样的错误。下面就谈谈Netscape中常见的错误信息及解决方法：
　　1.错误信息为“There was no response.The Server could be down or was not respondiry. If you are unable to connect again later,contact the servers admiristrator”。这是最常见的错误之一，特别是在高峰时间接入一些著名站点时常遇此错误。原因就在于时间和站点的问题，表明此服务器接入人数过多或者此页面已经失效。解决方法当然是避过高峰期，在人少时进入此网站。还有一种消极做法就是不停地向该网站发出连接信息，直至连上为止。对于页面失效则可以重新刷新一次。
　　2.404错误，即“Error code 404”。这也是在NetScape中常遇到的问题。往往是输入了错误的URL(地址)造成的，但也可能是此页面的链接不存在或更改了名字，如果是这样的话可以询问此页面版主，或者试试去掉子路径、在文件路径后加“/”等方法解决。
　　3.错误信息为：Netscape is unable to locate the server www.×××.com and try again”。此信息表明Netscape找不到此URL，即URL根本就不存在。这种问题只能从自身找原因，查看一下你的资料和资料来源是否正确，输入的字母是否正确。
　　4.出现断裂图像或没有图像的图标。这种错误分为两种情况。一是你在Netscape中关闭了图像传输功能。这样做无非是想提高浏览速度，但也同时使网络的丰富资源大打折扣，至于此问题的修改，很多资料均有详细说明，在此不再复述。再一种就是因为图像传输过程中出现数据丢失，造成传输失败。解决方法是点击该图标重新下载。
　　上面一些问题也许都容易解决，但最棘手的问题莫过于什么提示都没出现，等待很长时间也没有什么反应，最后出现了超时操作。面对这种问题，如果窗口底部的状态行的速度条移动缓慢，就应立即停止浏览(单击“停止”按钮)，以免白白浪费时间和金钱。
(李海华　王全)
