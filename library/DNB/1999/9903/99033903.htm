配置VFP的运行环境

　　在Visual FoxPro中，有时为了使用上的方便，需要按照自己的要求对VFP的系统运行环境进行适当的配置。本文就其相关参数及使用目的说明如下：
　　EDITWORK path：指定文本编辑器放置工作文件的位置。如果编辑的工作文件很大时，可以指定一个具有较大空间的位置。缺省位置是启动目录。
　　INDEX extension：指定Visual FoxPro索引文件的扩展名。缺省扩展名是.IDX。
　　LABEL extension：指定Visual FoxPro标签定义文件的扩展名。缺省扩展名是.LBX。
　　MVCOUNT：设置Visual FoxPro可以含有变量的最大数目。这个值的范围是从128到65,000；缺省值是1024。
　　OUTSHOW ON | OFF：终止启动通过同时按住Shift ＋ Ctrl ＋ Alt键隐藏所有窗口的能力。缺省设置为ON。
　　PROGWORK path：指定Visual FoxPro储存式程序的高速快取内存文件的位置。为了得到更快的效能，特别是在多人使用环境中，可指定一个快速的磁盘，允许为高速快取内存指定最小256K的空间（但是，文件可能会变得很大）。缺省位置是启动目录。
　　REPORT extension：指定Visual FoxPro报表定义文件的扩展名。缺省扩展名是.FRX。
　　RESOURCE path [\file]：指定FOXUSER来源文件的位置。file参数是可选的；如果不包含这个参数，Visual FoxPro就寻找FOXUSER.DBF文件。如果指定的文件不存在，则建立该文件。缺省值是启动目录（path）和FOXUSER.DBF（file）。
　　SORTWORK path：指定SORT和INDEX这样的命令存放工作文件的位置。由于工作文件会比储存的大两倍，所以需要指定一个具有足够空间的位置。为了得到更快的效率，特别是在多人使用环境中，可指定一个快速的磁盘（例如本地磁盘）。缺省位置是启动目录。
　　TEDIT [／N] editor：指定当你使用MODIFY COMMAND或MODIFY FILE命令编辑程序时，所使用的文本编辑器的名称。包含可选子句／N和TEDIT，可以指定Windows文本编辑器。缺省编辑程序是Visual FoxPro编辑程序。
　　TITLE title：指定在Visual FoxPro桌面窗口的标题栏出现的标题。缺省标题是[Microsoft Visual FoxPro]。
　　TMPFILES drive：如果在其他选项中没有指定，则本项指定储存暂存的EDITWORK、SORTWORK和PROGWORK工作文件的位置。由于工作文件会很大，所以需要指定一个具有足够空间的位置。为了得到更快的效率，特别是在多人使用环境中，可指定一个快速的磁盘。缺省位置是启动目录。
　　注：在组态文件中也可以使用SET命令、系统变量、COMMAND命令和STARTUP设置。
