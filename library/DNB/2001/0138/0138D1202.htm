3COM路由器故障排除

笔者所在单位与下级行连网使用帧中继线路，使用的路由器是3COM NETBuilder Remote Office224，此路由器具有固定的配置端口，它包括一个控制台CONSOLE端口主要用于路由器的配置，一个Ethernet端口用于连接局域网即1口，3个WAN同步端口，即2、3、4口，接口标准分别为V.35、RS-449及RS-232，目前使用的是1口和2口。1口连接本地局域网，2口连接的是128KB的帧中继。
故障现象：最近此路由器出现了故障，故障现象是打开路由器电源开关后，不能正常启动，前面板的Test灯一直闪烁，反复开机都是此现象。
故障分析：由于前面板的Test灯一直闪烁，说明路由器一直在检测系统硬件，在检测过程中发现有问题的硬件设备，检测不过去后一直停在测试状态。因为路由器不能启动，所以使用TELNET 命令无法登录到路由器，于是使用控制台端口进入路由器查看检测信息，此办法是以终端的方式连接路由器的CONSOLE口进行配置。
首先找一根CONSOLE通信线，此通信线用以连接微机的串行通信口(COM1)和路由器的控制台口(CONSOLE)，微机上的COM1口一般为9针公口，路由器的CONSOLE口也是一个9针公口，通信线的内部连接线是第2根、第3根进行交叉，第5根直连，一般随设备附有一条符合要求的通信线。
找到CONSOLE通信线，一端连接到微机上的COM1口，另一端连接到路由器的CONSOLE口，注意将两端都固定好，进入Windows98操作系统下，单击“开始”按钮，选择“程序→附件→通讯→超级终端”(如果没有超级终端选项，则可以在“控制面板”中的“添加/删除程序”内进行添加)，双击“Hypertrm”图标可进入超级终端，新建立一个连接，连接使用通讯串口1，通信参数设置为：速率9600bps；数据位8；奇偶校验 无；停止位1；软件流控Xon/Xoff，注意速率一定为9600bps，否则会出现乱字符或连接不通。这时打开路由器电源，在超级终端窗口中出现下面的检测信息：
3Com Corporation SuperStack II NETBuilder - SPMON Version 1.0.13
8192 Kilobytes of RAM installed.
2048 Kilobytes of Flash installed.
Flash file system tests passed.
EEPROM checksum test passed.
HSS self tests passed.
Ethernet Controller test passed - Station Address - 08000225DA10
SCC Internal loopback tests passed.
Memory tests
Ram March test Failure Expected 0x55555555 received 0x75555555
at address 0xBE5954
Access length = long word 32 bit .
Access type = write.
Entering Write/Read Scope...
To exit scope loop Cycle power or press the NMI switch...
从检测信息可以看出路由器中有8MB内存，加黑内容“Ram March test Failure”说明是内存检测时失败，反复启动路由器时都显示此内容，可以肯定内存有问题。
故障排除：关闭路由器电源，使用螺丝刀取下路由器的固定螺丝，然后打开路由器上面的盖板，可以看到在路由器的主板，在主板右侧有一个插槽，插有一条72线的4MB内存条，因为出厂时路由器所装的软件存在计算机Y2K问题，为了解决计算机Y2K问题，必须升级路由器软件，升级软件需要8MB内存，原来主板上只有4MB内存，此内存条是为了升级路由器的软件而加上去的。于是小心取下内存条，换上一块好的内存条(注：内存条是专用的，不能使用个人计算机中的内存条)，固定好上面的盖板，重新启动路由器，能顺利通过测试，接着引导系统文件，进入用户登录状态，路由器处于正常运行状态，故障排除。
(李志峰)
