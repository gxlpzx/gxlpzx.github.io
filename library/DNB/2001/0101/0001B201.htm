与Ghost比美的"拯救者"

    “拯救者”是由联想公司开发的一款强大的系统数据备份和恢复工具，可在联想主页http://www.lcs.legend.com.cn/service/drive/index.html下载到试用版。它具有全中文友好、简洁的视窗界面，并且结合DOS操作模式(支持鼠标操作)。它可以将用户的文件目录和逻辑分区静态备份到硬盘的“隐藏分区”或“普通分区”中，生成一个备份文件。如果用户的文件数据、系统遭到破坏，就可以用备份文件进行恢复。还可利用“拯救者应急盘”和“提取的硬盘分区信息”找回失落的逻辑驱动盘。

    拯救者与Ghost的对比 相同点：

1.    都具有快速的“分区备份”和还原功能(在同一个硬盘中)

2.    都有DOS和Windows两种操作界面(新版的Ghost) 3.对备份的数据都支持压缩功能

    不同点：

1.    各具特色的功能—Ghost具有硬盘克隆(复制)、分区、备份、网络备份等功能，偏重的是复制、备份功能；而拯救者所具有的备份(特别是“隐藏分区备份”)和还原功能，虽然也偏重于备份，可更多的还是安全恢复功能。

2.    备份文件的安全性—拯救者的隐藏分区备份功能可以将文件备份到“隐藏分区”中，而且可以设置密码，安全性相对较高；而Ghost制作的是镜像文件(.gho)，对于操作系统是可见的，所以可以进行如删除这样的操作，缺乏安全性。

3.    对多硬盘的支持—Ghost可以实现硬盘之间和网络之间的克隆(Local/Disk/To Disk)，而拯救者只支持对一个硬盘的操作。

4.    友好的中文界面—因为拯救者是国产软件，所以具有良好中文界面；这一点Ghost就没得比了。

5.    不同的适用范围—个人认为，拯救者和Ghost的分区备份功能适合于个人、商业等用户使用；而Ghost的“硬盘克隆”和“网络间硬盘克隆”功能更适合于公司、企业等用户使用。

    总的来说，“拯救者”与Norton Ghost二者各有特色，只有部分内容(如分区备份、还原)具有可比性。 使用实例： 下面以“拯救者”的标准版为例，介绍其使用方法。

    在Windows，先运行拯救者，进入视窗主界面(如图1所示)，这里有创建启动盘、隐藏分区管理、提取和恢复分区信息、还原和备份五个主要按钮(在DOS窗口中，有隐藏分区管理、分区备份、数据还原、特殊控制等按钮)，相应的使用方法如下：

1.    隐藏分区的创建与管理 “隐藏分区”是指硬盘中除主分区和扩展分区以外创建的分区，对于操作系统是不可见的，所以不能进行如复制、删除这样的操作，比较安全。它的(文件系统)类型是用“Hidden”标识的(在下面提到的“分区选择”窗口可以看到)。可以用来创建“隐藏分区”的包括：扩展分区(如D盘、E盘......)中的“未使用空间”和硬盘中除了基本分区(C盘)和扩展分区外的“未分区空间”(可以用FDISK命令来查看，选择第4项，如果在“Display partition information”中的Usage之和或者在扩展DOS分区的Usage不足100%，说明有未分区空间，否则，没有，一般是没有的)。 如果要创建“隐藏分区”建议使用大容量的硬盘，且连最后有一个不用或很少用的分区如E盘、F盘等,先将这个分区的数据移到其他分区，再用FDISK删除这个分区，人为地创造“未使用空间”。接着，在图1的界面中，我们选择“隐藏分区管理”按钮，系统自动重启动后进入DOS模式(如图2所示)，单击“创建”命令进入“隐藏分区管理”窗口，在最大值、最小值之间输入分配给隐藏分区的容量值(假设为384MB)，确认之后即可创建一个隐藏分区，这也是实现“隐藏分区备份”之前的必要操作。选择“卸载”命令可以将已创建的隐藏分区及备份文件删除并释放占用空间，转化为“未使用空间”(不能用来进行正常的读写操作，只有通过分区工具如FDISK对其进行定义后才能使用。)；“删除”命令可以将已建立的备份文件删除。建议使用之前最好先制作下面将要提到“应急盘”和提取硬盘分区信息，以防不测！

2.    备份 “拯救者”的备份功能可以将数据文件和分区硬盘备份到“普通分区”或“隐藏分区”中，同时提供备份压缩和备份加密功能，分“分区备份”和“目录备份”两种： 分区备份——可以将一个逻辑分区(如C盘、D盘、E盘……)进行完整的备份，对于操作系统所在的分区进行备份是十分必要的，相当于Norton Ghost的“Partition”(分区备份)功能。我们以备份C盘为例，首先，我们点击“备份”按钮，在图3所示的“备份方式选择”窗口中选择“分区备份”，之后在如图4的“选择要备份的分区”(我们选择C:\)，选择存放的目标分区(如果已建立隐藏分区，可以选择隐藏分区；如果没有可以选择C:\、D:\或E:\等这样的普通分区，但是要保证目标分区有足够的剩余空间)，这里还可选择是否备份压缩、只备份该分区中的有用数据、对备份文件加密(密码必须小于或等于八个字符)。再点击“下一步”对备份的文件取名(最多八个字符，默认backup1)，可输入备份文件的相关说明文字。最后，点击“确定”，电脑将重启动后退到DOS模式，并显示“是否现在开始备份”，选择“是”开始进行备份。在备份进程条中，我们可以看到备份的“传输速率”和进度，备份完成后，会显示“备份完成”窗口，确认即可。这时，C盘的备份才算结束。如果是备份到普通分区中，我们在Windows中可以找得到(以.res为扩展名)。 目录备份——可以将用户选择的目录(一次最多可以选择10个目录进行备份，这些目录可以不在同一分区中)备份到“普通分区”或“隐藏分区”中。目录备份的操作过程与分区备份差不多，这里不再介绍。

3.    还原 这里我们还是以上面提到的对C盘的分区备份为例，假设备份文件叫“backup1.res”,已备份到隐藏分区中。如果系统遭到破坏，我们就可以通过“拯救者”的还原功能迅速地进行系统恢复。具体操作如下： 首先，在图1的主界面中选择“还原”，重启后进入纯DOS模式，在如图5所示的“分区选择”窗口中列出当前分区情况：1表示C盘，2表示D盘，3是本人硬盘中的隐藏分区，容量中列出了各分区的大小(本人使用的是4.3G的硬盘)，分区类型中列出了当前的文件系统类型(除了熟悉的FAT32、FAT16外，还有一个Hidden，这是隐藏分区的特有类型)。我们选择存放备份文件所在分区，这里选择3(隐藏分区)，确定之后会进入该分区现有的备份文件列表(有文件名、备份日期以及相关文字说明)窗口，选择相应备份文件(backup1.res)点击“确认”(如果设置了密码，会要求输入密码验证)，这样就开始还原，当还原完成后，会显示“还原完毕”窗口，同样“确认”，这时你的系统已经完全恢复了。

4.    创建应急盘 先选择主界面的“创建启动盘”按钮，再插入软盘，点击“确定”按钮即可制作出启动盘。它的工作原理就是先对插入的软盘进行格式化，再拷贝必要的文件到软盘。这里的“启动盘”不同于我们熟悉的系统启动盘，而是“拯救者”自身创建的应急盘。虽然它的体积比系统启动盘小了很多，你可别小看了它，它可以在系统崩溃时恢复系统，而且结合备份的分区信息可以将丢失的逻辑驱动盘找回。

5.    提取和恢复分区信息 单击“提取和恢复分区信息”按钮，还是要退出到DOS模式，进入“特殊操作”窗口(如图6)。通过“提取”命令可以将当前硬盘的分区信息存储到指定路径的备份文件中(系统默认的是“A:\Fengqu.pat”)；通过“恢复”命令可以将选定的分区备份文件恢复到备份时的硬盘分区结构状态。建议将应急盘和分区信息文件存放到一块儿。 注：在1.0“试用版”中，视窗界面没有以上所述的第1项和第5项，但是在DOS窗口中，是有的，而且与标准版的基本功能相同，只是窗口略有不同。

注意事项：

1.    硬盘必须有足够的空间才能使用“隐藏分区备份”功能，但是，“隐藏分区”不能创建在C盘中，如果隐藏分区创建在这样的普通分区中，会对该分区的数据和隐藏分区本身的数据有所影响。而且，每个硬盘上只能有一个隐藏分区，而每个隐藏分区只能有两个备份文件(一个可以被覆盖，一个不能被覆盖)。

2.    在进行备份时，不能通过“目录备份”方式备份根目录，只有选择“分区备份”方式才可以。而且最好不要对“Program Files”和“Windows”目录进行目录备份和恢复。如果备份文件存放的目标分区是普通分区，那么所取的备份文件文件名不能与已经存在的备份文件重名。而且，进行目录备份的文件夹，建议尽量使用英文名，且易短不易长。因为拯救者是以DOS文件命名方式命名文件夹、文件名的，所以如果同时还原的几个前8个字符名称相同的文件夹且又在同一目录，会造成同名相互覆盖。

3.    在进行还原操作时，会破坏目标分区或目标目录现有的数据，我们应该谨慎使用；在进行恢复硬盘分区信息操作中，如果备份文件的硬盘分区信息结构与当前硬盘的分区信息结构不相同，那么当前硬盘上现有的全部数据将有可能丢失。在备份和还原的过程中不要随意中止进程，否则可能会导致数据的丢失。

4.    对备份文件的说明文字最好使用英文或拼音字母，否则在DOS模式下不能识别。

5.    制作应急盘时，插入的软盘的现有内容将被删除。
