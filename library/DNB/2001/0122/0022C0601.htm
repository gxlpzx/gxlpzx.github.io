稳定压倒一切——Hot CPU Tester轻松学

Hot CPU Tester是一款专门测试系统稳定性的强力测试工具，它可以帮你测试CPU超频后的稳定性，对于喜爱超频的朋友来说是再合适不过了，如果CPU超频后能通过Hot CPU Tester的连续测试，那么你就可以放心地使用了。
软件下载和安装
Hot CPU Tester是一款共享软件，目前的最新版本是1．1．2，软件大小只有507KB左右，可以在Windows 9x/Me/NT/2000各种操作系统平台下使用，你可以到华军软件园(http//bj．onlinedown．net/HotCPUTester．htm)或者Hot CPU Tester的“老家”(http//www．opusware．net)下载该软件的最新版本。下载后得到一个名为hotcpu．exe的可执行文件，直接运行它就可以进入安装程序，选择好安装路径后，一路“Next”就可以了。安装结束后，Hot CPU Tester会在桌面和开始菜单中自动生成快捷方式。
软件使用指南
运行Hot CPU Tester后，你首先看到的是该软件的版权信息，程序在这里告诉我们测试版本只能用来测试MMX功能，如果想测试SSE、3DNow、Chipset、Memory、L1/L2等，那么需要购买增强版本。不管它，点击“OK”按钮进入如图1所示的程序主界面，下面笔者就详细介绍该界面中的主要项目。
1．“Testing parameters”(测试参数)：你可在该栏目中选择测试的时间(未注册版本只提供1小时、3小时、6小时三种测试时间选择，如果是Pro版本则还有12小时和Manual可供选择)，一般情况下只需在1小时、3小时、6小时三种选项之间作出选择，如果你想测试服务器的稳定性，那么可以选择12小时或更多的时间，然后点击“Run”按钮便可开始进行测试，在测试过程中可以随时点击“Stop”按钮停止测试。
此外，“Testing parameters”栏目中还有一个Hardcore复选框，它是专门为700MHz以上的CPU或者多CPU设计系统的，只有你的CPU频率大于700MHz才可以进行此项测试，反之，程序将关闭此项功能(默认)，因为如果强行进行测试的话可能导致系统崩溃。
2．“Console”(控制台)：该栏目将显示Hot CPU Tester在测试过程中发现的错误，点击这一栏中的“Analyze”按钮还可以了解到更详细的信息。不过，只有当Hot CPU Tester发现错误时此按钮才呈可用状态。
3．“CPU Quality Monitor”(CPU状态监测)：在测试过程中，你可以随时从“CPU Quality Monitor”监视图中观察到CPU的工作状态，正常情况下该栏目值应该是在90%～100%之间(如图2)，某些情况下可能会下降到20%以下，这可能是有其它程序正在同时运行或者Windows系统出现了问题。不过，如果多次出现这种情况，那么说明你的CPU可能有问题。
4．“Modules information”(模组信息)：在该栏目中你可以看到CPU的Frequency(时钟频率)、Fibonacci、Prime、Calculating、Complex Matrix、Chipset(芯片组)、L1、L2、Memory(内存)等信息。常见的错误提示信息有：Error in Math instructions(Math模块有错误)、Error in Matix instructions(Matix模块有错误)、Error in Fibonacci instructions(Fibonacci模块有错误)、Error in MMX instructions sets(MMX模块错误)、Error in SSE instruction sets(SSE模块错误)、Error in 3DNowinstruction sets(3DNow模块错误)Error in L1 Cache(L1内部缓存错误)、Error in L2 Cache(L2外部缓存错误，可从CMOS中进行检查，因为某些主板有ECC校验功能，请关闭有关的设置)、Error in system Chipset(系统芯片组错误，如果不是超频原因，那么最大的可能是主板有问题或者CMOS的设置有问题)、Error in memory(内存错误，虽然Host CPU Tester并没有检测内存错误的功能，但CPU、芯片组方面的错误也会导致内存出错。换句话说，如果看到这行错误信息，那么可以肯定CPU、芯片组方面有问题)。
总的来说，Host CPU Tester的使用还是比较简单的，除了测试时间稍长以外，上手还是很容易的。(阿志)
