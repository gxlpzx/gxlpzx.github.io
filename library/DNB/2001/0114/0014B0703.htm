{蓝点Linux 2.0下不能调出输入法状态栏问题的解决}

{ Blinp输入法是蓝点Linux 2.0自带的输入法，它同时支持XIM和Cxwin规范，对于国际化的程序支持光标跟踪，默认热键兼容Windows环境。现在暂时支持全拼，双拼，五笔和自然码输入法。可是笔者最近在安装蓝点Linux 2.0后进入Xwindow下，使用“Ctrl+空格”组合键却不能调出输入法状态栏，原来这是蓝点Linux 2.0的一个小Bug.不过还是让笔者找到了解决的办法。
方法一：先删掉/usr/X11R6/bin/chinput这个文件，
# rm /usr/X11R6/bin/chinput
再安装光盘(安装盘)上blinp开头的三个软件包。
blinp-0.1-4BP.i386.rpm
blinp-ccetab-0.1-6BP.i386.rpm
blinp-kde-0.1-1BP.i386.rpm
例如安装blinp-0.1-4BP.i386.rpm这个软件包，在控制台下键入命令
# rpm -i blinp-0.1-4Bp.i386.rpm
方法二：先删掉/usr/X11R6/bin/chinput这个文件，再安装光盘上cxwin-tools开头的软件包。
下次再进入KDE环境时，就可以在右下角的面板上看到一个类似Windows下的输入法图标，按“Ctrl+空格”就可以打开/关闭中文输入法；“Alt+1”“Alt+2”“Alt+3”可以切换当前内码(EN/GB/BIG5)；“Ctrl+Shift”切换输入法；“Ctrl+Esc”可以隐藏输入条；“Ctrl+.”可以切换全角标点。
注：笔者使用的是蓝点Linux2.0,内核版本2.2.16.
}
