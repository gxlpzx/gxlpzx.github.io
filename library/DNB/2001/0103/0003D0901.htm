{网络配置中的几个模糊概念}

{ 今天,随着Internet的发展和网费的下调,越来越多的人开始了自己的网上旅程,在Internet中驰骋的感觉不言而喻,但要你认真仔细地配置一下网络参数,你可能会发现自己对Win98、WinNT中的很多网络设置都是一知半解,比如：RTS/CTS流控、XON/XOFF流控制、DNS、MNP5选择……反正大概知道选择哪个,但是具体为什么要这样就弄不清楚了。正所谓：“知其然,不知其所以然”。其实把这些设置弄懂,不仅是电脑学习的深入,也是自己“快猫加鞭”提高上网速度和节省大家口袋里银子的好办法。
一、 关于调制解调器的数据压缩和纠错协议设置
Modem应该是大多数网民最熟悉最钟爱的小东东了,对于它的设置也有很多由来和学问,当我们打开“控制面板”→“调制解调器”→“属性”→“连接”→“高级”选项时,在高级连接设置中,有两个框架,先看左边的“使用差错控制”,Modem的速度从以前的14.4K到现在的56K,提高了近4倍,速度的提高既是好事但又带来了一个负面的影响,那就是传输率的提高会使传输的误码率大大增加,特别是在电话线路不好的情况下,更为突出。因此CCITT(国际电报电话委员会)和MICROCOMG公司分别制定了V.42bit协议和MNP4、MNP5(Microcom Network Protocol)协议。V.42和MNP使用滤出线路噪声和自动重传错误数据的技术来克服电话线路的信号损失,使误码率降到几乎为零,因此在这里要把“使用差错控制”选中以确保数据的完整性；另外,V.42和MNP5协议还对传输数据有压缩作用,就像用ZIP或RAR压缩文件一样,V.42能将传输文件最大压缩到4比1,MNP5能以2比1压缩,而且它们都将传输的数据以8位比特发送,比普通的10位比特发送(多了一位起始位和终止位)快了20％,当然压缩的比例与接受和发送的数据格式有关,比如像JPG、ZIP这样的文件压缩比几乎为0,但对TXT、BMP这样的文件压缩比可达到2比1 ～ 4比1。因此压缩数据项是必须选定的。
二、 关于调制解调器的流控设置
打开“高级连接设置”对话框,使用流控制是一个很重要的网络设置,它是计算机控制Modem的方式。在“硬件”选项中,有两个标识RTS/CTS,它们能起什么作用呢？原来计算机通过设置RTS信号为低电平0来停止Modem的数据传输,设置为高电平1时使其恢复数据传输；CTS是硬件通过设置CTS为低电平0来暂停计算机的数据发送,设置为1时恢复发送,相当于录音机的PAUSE键。在下面的“软件”选项中的XON/XOFF,作用与硬件大致相当,软件流控不使用RTS和CTS信号,而是发送字符XOFF来停止数据的传送,发送XON字符来恢复数据的发送,其本质完全靠计算机软件(驱动程序中内置)来完成的,软件流控不适合于用来传输二进制文件,因为它会把文件中的数据误认为是流控信号,再有其响应速度也比硬件慢,因此,这里的设置建议选中硬件流控。
三、 关于TCP/IP属性中的DNS设置
打开TCP/IP属性对话框中的DNS配置标签。很多朋友在这里的设置都是空白,也不知道DNS究竟有什么作用。其实DNS即Domain Name System(域名系统)。首先,要知道在互联网上每一个域名都对应一个唯一的IP地址(实际上域名的出现就是为了简化记忆IP地址),当你上网的时候,每当敲入一个网址,例如：http://www.163.net ,你的DNS服务器(一般是你的ISP提供)就会将这个网址转换为诸如：202.108.255.201这样的IP地址,其搜索顺序是“由大到小”,即：net→163.net→www.163.net ,最后如果查证有该IP地址,就开始登录此IP所指的网页。因此该标签的设置关系到你上网的速度,在不填的情况下,DNS为默认的指向你ISP的DNS服务器,但你有时会发现上网速度很慢,特别是“黄金时段”。这主要是因为：网络拥挤使上网的带宽变得拥挤,再有就是你的DNS服务器处于高峰访问期间,你的请求会在队列中等待处理。因此,为了加快你上网的速度你可以把DNS服务器设置为本地的或者那些速度又快又很少人知道的服务器。具体的“DNS Server”(域名解析服务器)IP地址可打电话或E-mail询问各大ISP服务商,如果有辅用DNS,可在主DNS后添加输入,这样会使上网速度进一步提升。
}
