{Squid使用技巧六则}

{ Squid是Linux下的一种代理服务器，它实现了对HTTP、FTP和Gopher协议的缓存。该软件功能强大，使用简单，易于上手，实乃Linux玩家的必备工具。下面是该软件在使用时的一些技巧：
一、安装后，因为Squid的访问控制列表默认是拒绝所有的请求，所以在第一次运行建立交换目录命令之前，需要修改/usr/local/squid/squid.conf配置文件。编辑/usr/local/squid/squid.conf文件，添加一行：
acl local＿net src 192.168.1.1/255.255.255.0
为了定义允许使用缓存的IP地址组，还需再添加一行：
http＿access allow local＿net
把这一行放在http＿access deny all的前面。
二、启动Squid后，为了验证Squid是否在工作，可查看目录/var/log/squid中的cache.log和store.log，它们能够表明Squid正在运行。
三、Squid并不限制用户访问任何站点，但管理员可以在squid.conf文件中自行定义访问规则，设置访问控制列表，然后根据这些列表允许或拒绝访问者的请求。不过大的ACL规则列表可能会降低负荷很重的Squid代理服务器的速度。
四、Squid会在代理服务器上建立日志文件，并把浏览过的Web站点保存在日志文件中。
五、因为Squid功能受到Squid.conf文件的控制，所以大家在使用Squid时应仔细掌握对该文件的调整编辑，以适合自己的需求环境。
六、Squid发行包带有一个cachemrg.cgi程序，其功能是能够以详尽的细节来描述代理的执行信息。大家可要好好利用噢。
如需要Squid的话，大家可从站点：ftp//squid.nlanr.net/pub上获得它的最新版本。}
