{软驱故障与刷新BIOS}

{ 好友的电脑坏了，刚开机自检，便停了下来，出现“FDD Controller Failure (80H)”提示。这不就是软驱设置有问题嘛，我先按下Del进入CMOS设置程序，仔细地检查了一遍设置，并没有发现问题。是软驱线松了吗？我打开机箱一看，软驱后面的数据线和电源线接得好好的，而且方向也没有接错。那么故障会不会出在软驱身上？于是我把软驱连同数据线拆下来拿到别的机器上试了一下，软驱完全没有问题。我又换了一个新软驱拿到这台机器上，故障依旧。
此后，我又屏蔽掉CMOS中的错误检测并重装了WINDOWS 98，其结果不是找不到PS/2鼠标、MODEN，就是在设备管理器中出现了许多乱七八糟的黄色惊叹号。此时，我开始怀疑主板了。于是我问了好友在电脑出现故障之前都干过些什么，他说他昨天在上网时，下载了一个最新MVP4主板的BIOS程序，并且用Aflash.exe程序将它刷新了。这时，一切问题均真相大白故障现象表现在软驱及其它外设，病因却在BIOS。
故障原因：这台机器是在刷新了主板的BIOS程序后，首先软驱无法使用，然后无法进入WINDOWS，勉强进入WINDOWS后，鼠标器和串并口都无法使用。这样我们不难发现整台机器的故障便是由于刷新了BIOS程序造成的。这台机器主板的品牌不明，只知道它采用的是台湾威盛公司的VIA MVP4芯片组，北桥芯片是VT82C694X，南桥芯片是VT82C686A，存储BIOS程序的是一粒型号为“SST29E01”的EEPROM电可擦除可编程存储器，而不是我们常见的Flash ROM(闪存)。众所周知，现在世界上生产微机主板芯片组的厂家就只有INTEL、ALI、VIA和SiS等少数几家而众多主板生产厂家所生产的主板，大多采用这些厂家生产的芯片组，所以我们在购买主板时一般都只关心主板的芯片组类型。这样就容易给我们造成一种误解，简单地认为只要主板所采用的芯片组相同，那它们的BIOS程序也一定相差不了多少，从而忽略了主板上另一个重要的器件——I/O芯片。不同厂家所生产的主板，不仅制造工艺不同，而且所采用的I/O芯片及电子元器件也不会相同。主板上的BIOS芯片所存储的就是这块主板的所有的硬件信息，所以不同厂家生产的主板，虽然采用的芯片组相同，但是它们的BIOS程序是一定不能通用的。如果用了别的厂家主板的BIOS程序来刷新主板，很有可能会造成电脑在启动后，在BIOS接管系统时，便无法控制主板上的硬件设备和I/O接口。现在可以断定这台电脑故障的原因就是由于升级BIOS程序时，用了别的厂家的BIOS程序而造成的。
故障处理：显然，只要把原来的BIOS程序写回去就能够恢复正常了。可是粗心的好友不但没有备份原有的BIOS程序，而且连主板的包装和说明书也弄丢了。这时，只好上网去找来了MVP4主板的BIOS程序(http://www.award.com.tw/download/bios)。因为把BIOS ROM插到电脑上后再用Aflash.exe去刷写太麻烦了，所以我去买了几个1兆的EPPROM，又借来了一台ROM编程器，然后我和朋友两人，一个写BIOS，一个插到主板上试。在我们将第0087号BIOS插到BIOS插座后，打开电脑，终于找到了软驱，在DOS下鼠标也恢复了正常，接下来赶紧格式化C盘，重装WINDOWS 98及驱动程序，一切都恢复正常，故障解决了。
经验教训：现在有许多的电脑玩家常常试着用一些品牌机的BIOS程序来升级自已的主板或显卡，从表面上看，一块杂牌的板卡转眼间成了名牌的主板或显卡，可是这样做不仅存在着很大的危险性，而且弊大于利。不同厂家生产的板卡不仅制造工艺不同，而且采用的接口IC及电子器件也不一样，他们所提供的BIOS程序理所当然也不能通用。我奉劝各位电脑玩家不要随便用不同厂家的BIOS程序来升级你的主板或显卡，只有使用原厂家所提供的BIOS程序才能最大限度地发挥主板的效能，升级BIOS时也要到厂家的网站或知名的驱动程序网站上去下载软件才安全。}
