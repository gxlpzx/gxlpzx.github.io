{NT下Windows98有盘、无盘共存}

{ 前一段时间Windows NT无盘站技术使得不少学校、培训班等大受其益。而随着计算机的普及，各种硬件也越来越便宜，于是，大多数用户便考虑升级系统。
我校于1998年购进50台工作站，全部无硬盘。2000年3月，由于课程需要，便把其中30台加了硬盘，装了WIN98操作系统(于是，整个网络便成了对等网和无盘网共存)。但随之而来也遇到了不少问题。下面，就笔者安装的体会，给大家谈谈应注意的几个问题。
首先，安装WIN98由于工作站配置相同，硬盘型号也一样，所以只需在一台机器上安装好WIN98就行了。其它则采用Ghost复制，这样可节省不少时间。
其次，要对每台WIN98工作站进行网络设置，这里应重点注意。因为是采取Ghost全盘复制的。所以，每台机器的设置都是完全一样。而当所有的WIN98同时打开时，由于每台机器原本使用远程启动，网卡上都带Rom芯片，则会出现网卡IP地址与Rom地址冲突，而无法访问服务器及网络，这时应关掉所有WIN98机器，然后再依次打开进行网络设置(具体设置跟WIN98对等网相同)。提示：在把硬盘装入计算机前，最好把所有的无盘站整理一遍。根据笔者经验，在添加无盘工作站之前，如果同时把所有工作站都打开，则在服务器远程启动管理器上，出现的Rom地址的排列跟机房机器的排列是不一样的。而为了更好的管理机房的微机，就应根据微机的编号依次间隔一定的时间开机，这样，就便于在“远程启动管理器”里，给无盘站编号，同时，这也给设置有盘站带来了极大的好处。
在这里，有几点应当注意：(以我校50台微机为例1～30号为有盘，31～50号无盘)
1.如果现在需要全部用DOS教学，则启动时，31—50号开机后，不用管它，而1—30号机当它 自检完成后会在屏幕左上方出现一排信息：
“Type H to boot from hard disk”
(注：这时如果按H键则系统将从硬盘启动，如果按回车键，则系统将从无盘启动。)
在这里，我们敲入回车，所有的机器将从无盘启动到DOS。
2.如果需要进行Windows教学，则当1～30号开机出现“Type H to boot from hard disk”信息时，按“H”键，系统就会自动进入Windows状态。与此同时，31～50号机照样可无盘启动到DOS系统。如果你的无盘站编号没有整理好，当有盘站与无盘站同时启动时，就会出现一些问题，主要有两种情况：
A.如果你将一台工作站从Windows启动，这个Windows系统在网络上的名称为01(假设)，而现在网络上已有一台编号为01的无盘工作站正在运行，那么在进入Windows系统时，则会出现：
“装载通讯协议编号时出错。”
错误38：指定的计算机名已经在网络上使用，请在“控制面板”选“网络”图标，指定其他名。
这时，我们只有另改它名。在另改其它名时，也应注意一点，就是：如果你改的名与无盘站名无关(假设“hong”)，则Windows登录Windows NT时，会出现如下错误：
“找不到用户名”
这时只需在NT服务器上的“域用户管理器”里添加一个新用户“hong”就行了。
B.如果你将一台工作站从无盘DOS启动，这个无盘DOS系统在网络上的名称为01(假设)而现在网络上已有一个名称为01的Windows正在运行，那么在进入DOS系统时，会出现：
NET 3056 A system error has occurred.
DOS Error 52 has occurred.
There is a duplicate name on the network.
NET 3546 Error running c\Lanman.dos\netprog\netwksta.exe.
Memory allocation error
Cannot load commandsystem halted
解决这种情况的办法就是给同名的Windows机器改名，或者在远程启动管理器里把这个无盘站编号改名。
由上面可知安排好无盘工作站编号的重要性。根据实际经验，在给无盘站加硬盘之前，先按微机的摆放顺序依次给无盘站编号(如：01、02、03……50)，而加上硬盘后，也按这个顺序给Windows编排名称。这样就不会出现上述情况，就算Windows什么时候突然坏了，也照样可顺利进入无盘DOS系统，而不必担心引起冲突。}
