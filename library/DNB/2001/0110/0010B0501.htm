{msdos.sys参数详解}

{ 众所周知，msdos.sys为Windows 9x的系统启动配置文件，但你是否知道其中一些参数的含义呢？让我来告诉你吧(以下内容以Windows 98为例)。
PATH部分
HostWinBootDrv=C
Windows 9x的主驱动器，默认值为C。
WinBootDir=C:\WINDOWS
Windows 9x的启动文件所在目录，默认为C:\Windows。
WinDir=C:\WINDOWS
Windows 9x的系统目录在安装时由用户设定，一般为“C:\Windows”。
注意：如果是在DOS或Windows 3.1安装Windows 9x并执行了备份系统文件，会有一项“UninstallDir=c”，它代表卸载Windows 9x所需的信息文件win95undo.dat和win95undo.ini所在的驱动器，默认值为C。
OPTIONS部分
AutoScan=1
当Windows 9x非正常关机后，下次启动时会自动运行磁盘扫描程序scandisk.exe。通过设置AutoScan的值可以控制scandisk.exe的运行方式。0表示禁用，1表示运行前询问，2表示自动运行。
BootKeys=1
表示开始引导系统时按“F5”或“F8”键是否允许出现菜单选择进入不同的模式。值为0表示允许，值为1表示禁用。
BootDelay=2
在Windows 9x启动时，当出现Starting Windows 98画面时可以按“F8”键来选择进入不同的模式。BootDelay定义了此画面持续的时间，默认为2秒。
BootGUI=1
表示进入Windows 9x图形用户介面(GUI，即Graphic User Interface)还是进入DOS，值为0表示进入DOS，值为1表示进入Windows 9x图形用户界面。如果想在DOS下进入Windows 9x图形用户界面，可以在DOS提示符下键入“win”命令来启动。
BootMulti=1
如果是从DOS或Windows 3.1升级Windows 9x，则可以在Winodsws 9x启动时按“F4”键选择进入旧版本的DOS或Windows 3.1。值为0表示禁用，值为1表示允许。
BootMenu=0
表示是否在启动Windows 9x时出现要按“F8”键才出现的那个选择菜单。值为1表示总是出现，值为0表示按“F8”键才出现。
BootMenuDelay=30
表示出现选择菜单的时间，默认为30秒。
BootSafe=0
修改BootSafe的值可以强迫Windows 9x每次是否进入安全模式。当值为0时表示让Windows 9x自动选择，当发生错误时自动进入安全模式，值为1是表示不管是否发生错误都进入安全模式。
BootWarm=0
表示是否显示进入安全模式时的警告。当值为0时表示不显示警告，当值为1时表示显示警告。
DblSpace=1
表示是否装载磁盘压缩驱动程序(Double Space)。值为0表示不装载，值为1表示装载，默认值为0。
DisableLog=0
控制是否记录引导过程的信息。值为0表示记录，值为1时表示不记录。
DoubleBuffer=0
0表示Windows 98不使用双缓冲，1表示Windows98自动决定是否使用双缓冲，2表示Windows 98使用双缓冲。
DrvSpace=1
表示是否装载磁盘压缩驱动程序(Drive Space)。值为0表示不装载，值为1表示装载。默认值为0。
LoadTop=1
LoadTop参数控制是否将CONNAND.COMDRVSPACE.BIN或BDLSPACE.BIN装入常规内存(640K)的高端。值为0表示装入高端，值为1表示装入低端。
LOGO=1
表示是否开启Windows 9x的蓝天白云欢迎画面。值为1时表示开启，值为0时表示关闭。
Network=1
控制Windows 9x启动后是否装载网络项。值为0表示不装载值为1表示装载。
WinVer=4.10.2222
表示所用的系统的版本号。
(广东 张习文)}
