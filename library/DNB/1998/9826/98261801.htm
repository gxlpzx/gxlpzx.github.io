如何用E-Mail访问WWW

　　E－mail 是 Internet 中使用最广泛的一项服务。它除了能收发电子邮件以外，几乎能访问所有的 Internet 资源。大家都很熟悉利用 E－mail 来实现 FTP，但很少有人利用 E－mail 来访问WWW，这是由于不同的WWWmail服务器有不同的命令格式，用起来比较麻烦。
　　但是对于那些手头拮据的上网者来说，这不失是一种省钱的好主意，另外对于那些只有 E－mail 功能而无 WWW 功能的用户来说，这更是一个好办法。它还特别适用于那些利用一些公共账号免费上网的用户，例如169，它对外有一个公共的账号：pub 或guest，该公共账号能够漫游该网内的大部分站点，但如果想访问因特网上的站点就不行了。而 E－mail 则没有这样的限制，它可以通达全世界任何地方。
　　下面以W3Gate。WwwFetch E－Mail Gateway两个Mail服务器为例介绍用E－mail访问WWW的方法(其中，方括号([])内的表示是任意可选的，可同时使用；尖括号(<>)内的表示是必须的；竖线(|)表示任选一个，但不能同时使用)。
　　一、W3Gate
　　信箱地址：w3mail@gmd.de
　　通过它可以访问WWW、FTP。你只要向上述地址发一封电子邮件，并在邮件正文中包含不超过10条 GET 命令行就行了。
　　语法：
　　[BEGIN | begin]
　　GET [options] URL | HELP | FAQ
　　[END | end]
　　你可以使用 HELP 来获得有关该服务器如何使用的帮助文件，也可以使用 FAQ 获得在使用该服务器过程中常见问题解答的帮助文件，而 URL 就是你想访问的网页地址，W3Gate 将以电子邮件的形式将该地址的主页送达你的信箱。命令大小写均可。允许最多有10条GET命令。
　　命令选项（options）：
　　[－t] [－u] [－a][－c columns] [－z] [－uu] [－img] [－l] [－s size]
　　－t : 除去所有的标签，取回一个纯文本
　　－u : 如果有可能将以相对路径保持所有的链接
　　－a : 以绝对路径保持所有的链接
　　－c columns : 一行显示的字符数
　　－z : 压缩文档
　　－uu : 在邮寄前对邮件编码（uuencode）
　　－img : 获取所有的相关的图片
　　－l : 获取所有的相关链接的文档
　　－s size : 设置发送文档的长度，取值范围1K～100K
　　如果URL 很长，一行写不下，可以用反斜线（\）来断行。
　　实例：
　　get －z －uu http://www.es.net/pub/rfcs/rfc1738.txt
　　get －s 16 －uu ftp://ftp.gmd.de/misc/computersongs
－1.4.Z
　　如果还有什么疑问的话，可以到它的主页上去瞧瞧，会有更加详尽的说明，不过是英文的，它的主页地址是http://www.w3gate.de/welcome.en.html 。如果未上因特网，也可以用W3Gate 来获取它，写一封邮件就行了，内容如下：
　　收件人: w3mail@gmd.de
　　主题:
　　正文：
　　begin
　　get －img http://www.w3gate.de/welcome.en.html
　　end
　　二、WwwFetch E－Mail Gateway
　　信箱地址：wwwfmail@wwwfetch.com
　　通过它可以访问WWW、FTP。如果要获得详细的帮助，向上述信箱发一封主题为help的信就行了，正文什么也不用写。
　　语法：
　　URL: <max 5 additional URLs separated by spaces>
　　Include: [Images,Executable, Archive,Sound,\ Movie, Postscript, Document ,Text, Compressed Files for Mac, Other ]
　　OS: [DOS | UNIX]
　　Files: <＃ of files <=5>
　　Limit: <＃ of MBytes <= 1.0>
　　Send: <No | MIME | UUE | TEXT>
　　Split: <size in KBytes>
　　注释：
　　URL:不超过5个URL，中间用空格隔开，最好是一行一个URL。
　　Include:指定下载文件的类型，可以多选，中间用逗号隔开。只有下列指定文件类型的文档才会被下载：Executable(exe、 com)；Archive(zip、 Z、 gz、 sea、 sit、 arj、 tar、 tgz);Sound(au、 aiff、 wav、 mid);Movie(mpg、 mpeg、 mov、 qt、 movie、 moov);Postscript(ps);Document(pdf、 doc);Text(text、 txt);Compressed Files for Mac(hqx);Other。
　　OS:确定返回文件名的格式。“DOS”格式（文件扩展名限制为3个字符，例如“index.htm”）或者“UNIX”格式（文件扩展名一般没有限制）。如果使用Windows 3.x 请选择“DOS”，如果是Windows95 或Mac使用者，可以选择“UNIX”。默认：“DOS”。
　　Files:取回的文件数，小于等于5。默认：2。
　　Limit:限制取回文件的长度（MB），小于等于1M，默认：0.2。
　　Send:选择发送文件的编码格式：“No”(不发送ZIP文件)；“MIME”(发送base64 ZIP的文件)；“UUE”(发送uuencoded ZIP的文件)；“TEXT”(发送文本文件)。默认：“No”。
　　Split:指定最大的分割文件长度（KB），最大：500KB。
　　注意：所有参数大小写均可。参数“URL”是必须的，其他参数可选。
　　实例：
　　收件人：wwwfmail@wwwfetch.com
　　主题：query
　　正文：
　　URL: http://www.wwwfetch.com/wwwfetch_pro/
　　URL: http://www.wwwfetch.com/wwwfetch/
　　Include: images,sound
　　OS: DOS
　　Files: 4
　　Limit: 0.5
　　Send: MIME
　　Split: 100
　　如果访问成功，你将会收到该服务器发来的一封带有附件的邮件，将附件另存为ZIP文件（这一步很重要），然后将ZIP文件解压缩到一指定目录下，然后用自己喜欢的浏览器对该目录下的index.htm进行浏览，该文档的界面非常友好，它会将所有下载的网址做成一个目录，只要单击超链接就可以浏览下载的网页了。
　　该服务器的主页地址是http://www.wwwfetch.com/wwwfetch/。
