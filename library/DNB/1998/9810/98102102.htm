无盘工作站上共享Windows及其应用软件

　　在使用无盘工作站时，常遇到一个问题：不能以386增强模式运行Windows，运行Word6.0和Excel5.0时会出现“内存不足”的提示。经过系统地分析并结合实际使用情况，我们采用下述的方法解决了这个问题，实现了无盘工作站上共享Windows和Word、Excel等软件。
　　网络环境
　　Netware3.11网络操作系统，六十台486－66/8M/1.2＋1.44工作站，两台Compaq Prosignia 4/66服务器，多网段(内桥连接)总线型结构。
　　安装与设置
　　1.在工作站上以Supervisor用户名登录入网，把Windows3.2、Word6.0、Excel50等安装(运行Setup)到服务器的F：盘上。
　　2.设定普通用户组(如STUDENT组)对Windows、Word、Excel等目录的权限为F(File scan)、R(Read)。
　　3.在Windows目录下建立swap子目录。设定STUDENT用户组对该目录的访问权限为S(Supervisor)。
　　4.在Windows\swap目录下建立swap1、swap2、swap3……swapN共N个子目录。每个swap后的数字对应一个工作站的编号，N为工作站的个数。
　　5.将Windows\win.com文件改名为wins.com。
　　6.修改系统登录文本(System login script)，加入以下一行:
　　MAP ROOT N:=SYS:WINDOWS\swap
　　建立N盘与假想根目录Windows\swap的连接。
　　7.在Windows目录下建立win.bat文件，内容为：
　　set temp=N:\swap％1(％1是每个工作站的编号)
　　wins
　　为每个工作站分配独立的磁盘空间存放临时交换文件。
　　8.运行Windows(执行\Windows\winscom)，在“控制面板/386增强模式/虚拟内存设置”项中，将N：盘设置为临时的磁盘交换空间，大小为默认值。在“程序管理器/选项”中，取消“退出时保存设定值”的设置，以免普通用户退出Windows时出现“＊＊＊组信息不能保存”的提示。
　　运行方法
　　以STUDENT组内的用户登录后，进入Windows目录，输入“win 工作站编号”即可运行Windows及Word、Excel等。例如：在10号工作站使用时，输入win 10。
　　性能评价
　　1.每台工作站的软件环境一致，支持所有工作站以同一用户名登录，适合公共电脑教学。
　　2.防止有意或无意破坏服务器上的文件，防止病毒的感染。
　　3.维护方便。只需定期清理swap目录下的因非正常退出而留下的临时文件。
　　4.不影响软盘的操作，仍然可以使软盘存取用户的文件或程序。
　　5.网络的通信量大，Windows的性能有一定影响。另外，由于使用远程启动，导致“文件管理器/磁盘/格式化”项不能正常格式化软盘。
　　6.对于有本地硬盘的工作站，在本地硬盘上建立临时或永久的交换文件效果会更好。
　　该网络在一年多的实际应用中，运行情况良好，可见，这是一个可行的解决方法。

　　（广东　陈航）
