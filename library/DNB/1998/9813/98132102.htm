NETWARE服务器随机存储器的扩展

　　去朋友单位时发现，该单位NETWARE服务器内存由16M升为32M，但性能没有多大改善，服务器硬盘仍然转个不停，究其原因发现其硬件内存方面虽然升级了，但NETWARE服务器内存设置没有任何改变。
　　NETWARE服务器最大支持4G的随机存储器，当随机存储器低于16M时，NETWARE系统自动进行管理，而大于16M时，需进行随机存储器扩展，也就是说即使用户在安装服务器时已安装好了大于16M的内存，而NETWARE也只能用上16M内存。
　　让NETWARE使用多于16M通常有两个办法：
　　1.建立DOS配置文件，在CONFIG.SYS其中加载设备驱动程序HIMEM.SYS。以硬盘起动为例：在C盘根目录下建立CONFIG.SYS文件，仅一行DEVICE=C:\DOS\HIMEM.SYS即可。
　　2.在控制台下使用REGISTER进行内存注册。以32M内存为例，具体方法是：在控制台下键入REGISTER MEMORY 1000000 1000000，前一个1000000是后16M的起始地址，后一个1000000是后16M的内存长度，可以在控制台下键入MEMORY命令检测注册是否成功，注册成功应显示：Total server memory:32,＊＊＊ Kilobytes。如果想长期有效，可将其写入AUTOEXEC.NCF中，即在控制台下键入LOAD INSTALL，选择System Options再选择Edit AUTOEXEC.NCF File，在load monitor命令行的前一行写入REGISTER MEMORY 1000000 1000000。退出服务器后重新启动服务器，这时你会看到服务器各缓冲参数都有较大提高。
