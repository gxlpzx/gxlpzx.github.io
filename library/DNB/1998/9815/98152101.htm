Novell和Windows NT混合网的组建

　　Novell网络和Windows NT网络是目前局域网中应用极为广泛的两种网络类型，这两种网络的单独规划和设计是大家较为熟悉的，但怎样将两者有机结合，互享系统资源，则是本文所要介绍的。本文所讲的“混合型网络”有两层含义，其一是指网络类型既包含Novell网也包含Windows NT网，其二是指网络的拓扑结构是总线型网络和星型网络的共存。文章以本单位所建的一个混合型网络为例，从网络的规划和选型、硬件的选择和连接、同时登录两个不同网络的工作站的设置等方面做以下介绍。
　　一、网络的规划和选型
　　1.网络的地理分布和用途
　　此混合网络主要分布在办公大楼和教学大楼两处，且办公楼与教学楼相距较近，仅有80米左右。教学楼设有一个计算机房，主要供学生上机使用，有机器60多台(其中无硬盘286、386微机10多台),早已联网；办公楼也设有一个计算机房，供教师使用，现共有486以上的微机15台。
　　2.混合局域网的设计和规划
　　综合考虑网络的用途、现有机器的类型以及网络的安全等因素，网络规划为：因学生用机的部分硬件设备较差，为了充分利用所有硬件资源，保留原有的网络系统。办公楼的15台微机因硬件设备较好，同时主要是用于计算机教师的软件开发和对其他教师的培训，因而将这15台微机联成一个小型局域网。当两个局域网分别规划好后，便可进行两者的混连规划。其网络的整体规划结构如图1：
　　3.网络的选型和连接
　　教学楼部分保留原有的NetWare 3.12网络系统，由一个网络服务器带4个16口的HUB，每个HUB接16台微机，60余台微机全部连入此网络；办公楼的15台微机组建成Windows NT网，15台工作站以总线型结构采用同轴电缆线串联在NT服务器上。同时，用双绞线将这15台工作站以星型结构连接在由NetWare网引出的一个24口HUB上(15个工作站采用24口HUB连接是为了便于将来的扩充)。这样，一个网卡便连有两条网络连线，同时整个网络的拓扑结构基本上是以总线型网络为主干，以星型网为分支的混合型网络。
　　二、网络服务器的安装
　　1.对NetWare 3.12服务器的改造
　　NetWare 3.12服务器已装有一个网卡，再连入15台工作站后，为了减轻一个网卡过重的工作负担，需再给服务器安装一个网卡，此网卡最好与原有的网卡同型号(16BIT，NE2000兼容卡)。为了能使一个服务器中的两个网卡都能同时工作，在将两个网卡安装好后，需修改服务器上的Autoexec.NCF文件，使每个网卡占用一个IRQ值，以防止地址冲突，而且不同的网卡需要有不同的Int与Port值。以下是安装两个网卡后的Autoexec.NCF文件的内容：
　　File server Name XYW
　　IPX Internal Net 199801
　　Load NE2000 Port=300 Int=3 Frame=Ether_802.3 Name=NCI1
　　Bind IPX to NCI1 Net=A
　　Load NE2000 Port=320 Int=5 Frame=Ether_802.3 Name=NCI2
　　Bind IPX to NCI2 Net=B
　　Mount all
　　Load Monitor
　　2.Windows NT网络服务器的安装
　　NT网络服务器采用一台Pentium 200 CPU、3.2G硬盘、64MB RAM的兼容机为服务器，网卡为32BIT双口(具有RJ－45接口和BNC接头)的PCI卡(带有NT4.0的网卡驱动程序)。网络操作系统为中文Windows NT Server 4.0。具体的安装过程请参考有关指导书，不过为了顺利地安装和维护，提以下几点建议：
　　(1)将“服务器类型”选为“主域控制器(PDC)”；
　　(2)虽然NT4.0自身提供了大量的网卡驱动程序，但为了能够正确安装，最好使用网卡自带的驱动程序；并记下网卡在没有冲突时的IRQ 和I/O的值；
　　(3)网络协议选择为NWlink和NetBEUI两种；
　　(4)输入并记下服务器的“域名”；
　　(4)为了日后网络服务器维护的方便，最好制作一张紧急修复磁盘。
　　三、NT工作站的安装
　　虽然可以由DOS、Windows3.x、Windows NT Workstation和Windows95等工作站登录到Windows NT Server 4.0服务器，但笔者在实际安装中认为Windows95可以说是NT4.0最佳的工作站操作系统。同时这15台NT工作站必须要同时登录到Windows NT和NetWare网络，因而15台工作站全部选择用Windows95登录。
　　1.工作站网卡的选择和连接
　　为了能同时登录到Windows NT和NetWare网络，15台工作站上的网卡必须是双口网卡，现选择市面上普通的32BIT网卡(如果要安装无盘工作站，网卡的选择要十分慎重)。每一个网卡需同时连两条通信线，其中BNC接头接入同轴电缆线上串接的BNC的T型头，而RJ－45接口用双绞线连到24口的HUB上。
　　2.工作站软件的安装
　　因选用Windows95连入两个网络，所以每个工作站先预装Windows95操作系统，在确认网卡等所有硬件设备无故障和冲突之后再安装Windows NT客户“Client for Microsoft Networds”和NetWare客户“Client for NetWare Networks”。
　　(1)安装Windows NT客户“Client for Microsoft Networds”
　　选择“开始 /设置 /控制面板 /网络”，出现“网络”对话框。单击“网络”对话框中的“添加”按钮，又弹出“选定网络组件类型”的对话框，选择对话框中的“客户”选项，再单击“添加”按钮，会弹出“选定网络客户”的对话框，在此对话框的“厂商”下选择“Microsoft”选项，在“网络客户”处选择“Microsoft网络客户”，单击“确定”按钮，返回到起始的“网络”对话框。选择“网络”对话框中的“Microsoft网络客户”选项，单击“属性”按钮，将出现图2所示的“Microsoft网络客户属性”对话框，此对话框用于设置工作站所能登录的Windows NT的域名(此域名在安装NT服务器时建立)，当输入正确的服务器域名后，按“确定”按钮，此设置结束。
　　(2)安装NetWare 客户“Client for NetWare Networks”
　　选择“网络”对话框中的“添加”按钮，弹出“选定网络组件类型”的对话框，选择对话框中的“客户”选项，再单击“添加”按钮，会弹出“选定网络客户”的对话框，在此对话框的“厂商”下选择“Microsoft”选项，在“网络客户”处选择“NetWare网络客户”，单击“确定”按钮，返回到起始的“网络”对话框。选择“网络”对话框中的“NetWare网络客户”选项，单击“属性”按钮，将出现图3所示的“NetWare网络客户属性”对话框，在图3对话框的“首选服务器”处选择工作站要登录的NetWare服务器名(此服务器名必须是NetWare 服务器名称，否则无法登录)，并设置第一个网络驱动器名代号。按“确定”后设置结束。
　　(3)关于登录的先后顺序的说明：
　　如果在“网络”对话框的“基本网络登录方式”选项中所选的是“Microsoft网络用户”，则开机后先登录到NT网；如果选择的是“NetWare网络用户”，则开机后先登录到NetWare网络。建议NT服务器所使用的“域名”与NetWare服务器所使用的“服务器名”相同，并且在两个不同的服务器上给同一个工作站相同的用户名，这样要登录哪个服务器只需在“网络”对话框的“基本网络登录方式”处选择不同的登录方式便可以了， 避免每次开机时输入不同的“用户名”和“域名”。
　　此混合型网络在安装好后的3个多月来运行正常。
