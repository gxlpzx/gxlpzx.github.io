32位操作系统将大行其道

      1997年，电脑用户新购买的计算机几乎都安装了Windows 95或其OEM版本（Windows 95 OSR2，这是Microsoft为OEM厂商提供的Windows 95的升级版本），随着Microsoft撤消了对Windows 3.x的技术支持，可以想象，原有的Windows 3.x用户也会越来越多地转到Windows 95这个16位与32位代码混合的操作系统。可以这样说，1997年是PC机操作系统由16位过渡到32位过程中重要的一年。1998年，随着Windows 95的第一个正式升级版（开发代号Memphis，俗称Windows 98)及Windows NT 5.0的发行，新一轮软件的升级由此开始，其目标就是推动PC机过渡到使用全32位的操作系统上。
　　Windows NT是Micosoft Windows操作系统系列中最新、功能最强的成员，是纯32位的操作系统，具有内部安全性好和网络功能强等特征。过去因其需要的硬件配置较高，用户成本增加，网络远未如今天流行等因素而“曲高和寡”。
据说，Windows 98可能在1998年第一季度上市，但Windows NT 5.0可能会拖到1998年底或是1999年初上市。在Microsoft的产品策略中，将来Windows家族产品都要共用相同的核心代码，即NT的核心代码。但目前为了照顾已有的16位软件及16位的设备驱动程序，从而开发出了Windows 95这种过渡性的产品，其升级版本Windows 98起着继往开来的作用，以便用户能够平稳地过渡到32位系统上。虽然，Windows NT 5.0与Windows 98的外观看起来更为接近（同Windows 95和Windows NT 4.0相比较），两者功能更为相似，但在核心代码上仍然不同。将来，也许在1999年或是2000年，就会出现统一后的Windows NT，Windows 9X也许就不存在了。
　　下面根据各种媒体上的介绍以及从因特网上查阅到的信息，向读者介绍一下，用户将会从这两种操作系统中得到哪些功能，以便升级操作系统时做到心中有数。
　　一、Windows 98有什么
　　1. 与Internet Explorer 4.0（以下简称为IE4）的紧密集成。这或许是Windows 98与Windows 95的最大区别。Internet Explorer 4.0已于1997年10月正式发布，如果用户已经安装了IE4，则对IE4为Windows 95带来的变化就已经有了直接感受。
　　2. FAT32文件系统。从Windows 95 OSR2版本开始引入了FAT32文件系统。FAT32文件系统支持2TB（1TB=1000GB）的硬盘，而当分区小于8GB时，每簇的尺寸只有4KB（作为对照，在FAT16中，当分区容量小于256MB时，簇的尺寸为4KB，而在分区达到2GB时，簇的尺寸达到了32KB）。这是FAT32文件系统优于FAT16文件系统的地方。
　　3. 采用了WDM（Win32 Device Model）——新的硬件驱动程序规范。在Windows NT 5.0中也采用同样的规范。利用这一规范，硬件厂商为Windows 98开发的设备驱动程序也能在Windows NT 5.0中使用。采用这一规范是Microsoft使两大Windows操作系统接轨所采取的重要步骤。使用这种新的设备驱动程序规范，Windows 98支持AGP（Acceleration Graphics Port，图形加速端口）、DVD（Digital Versatile Disc数字多功能光盘）以及USB（Universal Serial Bus，通用串行总线）和IEEE1394端口（又称FireWire），使PC向家电产品靠拢的速度加快了。虽然采用了新的硬件设备驱动程序规范，但Windows 98仍然保持与老式的设备驱动程序的兼容性，甚至原有的16位的驱动程序也能在Windows 98下运行。
　　4. 采用ACPI（Advanced Configuration and Power Interface）——新的设置与电源管理界面。Windows 98不仅支持Windows 95中使用的APM1.2（Advanced Power Management，高级电源管理），而且还引入了ACPI。ACPI与APM不同，后者是由系统BIOS来决定何时停止硬盘的转动或是关闭显示器的显示，而前者是由操作系统来执行所有节能功能的。操作系统可在更为广泛和复杂的过程中发挥作用。ACPI还是Windows 98中另一创新——OnNow的基础。OnNow功能使得PC机可以像家电产品那样使用，打开电源，不必经历几分钟的引导过程就能立即使用。关闭电源之后，系统仍以极低的功耗保持通电状态，而一旦有外来信号（如发来的传真等），可以立即“唤醒”。
　　5. Help Desk——新的帮助系统，也可称为技术服务组件。它面向三种信息资源：本地的帮助、技术支持和升级管理程序。在Windows 98中所有的帮助信息都是基于HTML格式的超文本。本地帮助主要是帮助一般用户解决Windows的有关问题。升级管理程序通过检查硬盘而监视所有已安装的驱动程序和软件，更重要的是，可以卸载导致问题的驱动程序；技术支持是特别为Beta测试者设计的组件。
　　6. 其它改进。在Windows 98中，所有改进虽然并不是翻天覆地的，但确实有所改善。如安装过程中，制作应急磁盘成了不可越过的步骤，而且应急盘中加入IDE CD－ROM的驱动程序。工具提示（ToolTip）可以识别窗口右上角上的最大化、最小化和关闭按钮，想一想还有许多已经使用Windows 95好长时间的用户并不了解这三个按钮的功能，就可以明白这一改变确实有用。
　　二、Windows NT 5.0强在哪里
　　Windows NT 5.0外表看起来虽然与Windows 98相似，有许多功能也十分相近。但内核仍然是不同的，而且两个操作系统的市场定位也不同。Windows 98定位于消费类应用，NT定位于企业级应用。在以下的叙述中，主要指出两者不同的方面。
　　1. 即插即用与电源管理功能。 这是NT所有版本中第一次支持的功能，也是迈向统一的Windows过程中关键的一步。这两种功能对笔记本机用户都是至关重要的，这为在笔记本机中运行NT打开了通路。笔记本机的电池可以持续更长时间，还可以热插拔PC卡。
　　2. NT 5.0包含的重大进步之一是Active Directory（活动目录服务）。Active Directory提供了一个在企业内部广泛地管理与访问地址簿与计算资源的方案。例如，利用Active Directory目录服务，用户可以启动网络上的应用程序而不必关心应用程序到底存在何处。
    3. 另一个新增的重要功能是IntelliMirror（智能镜像）。一个“总”计算机（通常是网络服务器）保存着所有的应用程序和用户档案，这样用户就可以使用公司中的任意一台计算机登录，然后得到“自己”的桌面和应用程序。在这种情况下，本地硬盘变成了一个缓存，每次在本地硬盘进行写操作时，写入的信息会在后台“流入"服务器并永久性的保存起来。
　　4. 改进了的NTFS文件系统。NTFS文件系统除了保持原有功能，如可指定压缩个别文件或文件夹，可对个别文件或文件夹设置口令等之外，新的NTFS文件系统可以识别FAT32分区，但是却不能在FAT32上安装NT，而NTFS分区上也不能安装Windows 95或是98。
　　5. 采用了新的安全机制。这是一种称为Kerberos的，由MIT（美国麻省理工学院）开发的更为通用的安全机制。
　　6. 一体化的存储。这是一种新的存储系统，既有文件系统的高速性能，又有数据库的强大功能。由于这个系统在寻找信息方面的杰出特性，它可以免除保存多个拷贝的必要。比如一个用户要向十个人发送一份巨大的Powerpoint演示，这个存储系统会聪明地把这十个人指向同一个存储拷贝，而不需要建立同一文件的多个副本。
　　综上所述，Windows 98 和NT 5.0的所有新功能都是基于32位代码的，不要指望可从16位平台上获得这些功能。1998年不仅是PC硬件不断更新的一年，而且也是PC机操作系统从16位平台向32位平台转移的关键性一年。对用户来说，升级的结果都是提高了操作系统的可靠性和易管理性。随着因特网在新的一年中更进一步的普及，可以想象得到，真正的分布式计算正向我们一步步地走来，也许不到2000年，我们就会看到一个统一的真正32位的Windows——这就是NT 5.0的后续版本。
