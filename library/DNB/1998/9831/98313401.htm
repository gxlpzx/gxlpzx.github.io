专家坐堂

　   
　　江苏读者孙勇问：最近我在Windows 98(Beta版)下安装MAGNARAM内存管理工具，使用一段时间后，发觉效果不理想，遂将其卸载。该软件自带Uninstall程序，所以我在Windows环境下点击“控制面板”，进入“删除/添加程序”进行卸载，一切正常。但在下次启动时，当屏幕出现“Start Windows...”时，显示:“Cannot find a device file that may be needed to run Windows or Windows application......press a key to continue”，当我按任意键后，Windows仍能启动，但速度较以前慢了不少，请问如何解决?
　　答：对于在Windows下卸载程序后出现系统启动不正常的情况，一般来讲都是卸载程序时操作有误造成的。确实，对于大多数Windows用户来讲，基本上都碰到过这种“请神容易送神难”的情况。Windows作为一个多任务的操作系统，其最大的特点是系统资源的共享，这不仅体现在硬件资源上，而且在软件资源上也是如此。从Windows 95开始，系统利用注册表对所有安装了的应用程序进行注册，以利于系统的整个管理，因此，Windows 95系统下应用程序的安装与卸载远比以前的DOS系统复杂。当然，应用程序的安装一般不需要用户太多的干预，根据安装程序的提示进行即可，但卸载程序则不这么简单，从Windows 95系统开始，程序的卸载大致有以下三种情况：一是应用程序安装完成后没有Uninstall选项，这种情
况大多存在于早期的Windows程序，对此种程序的卸载，就笔者自己的经验，似乎没有什么好的办法。第二种是程序安装完成后虽没有Uninstall项，但通过Windows 95系统的卸载功能可进行卸载，也就是在系统的“删除/添加”项中列有安装完成的程序的名称。第三种情况就是在安装完成后有Uninstall选项的应用程序，但这种程序在系统的“删除/添加”项中也仍有注册，那么这种程序的卸载最好利用它自身的Uninstall进行卸载，因为系统提供的“删除/添加”项主要是针对一些通用的情况进行卸载，而对一些较为特殊的应用程序则不
一定实用，可能会造成卸载不彻底或有误的情况。从该读者所述情况看，正是由于没有利用应用程序本身的Uninstall功能进行卸载，而选用了系统的“删除/添加”项进行卸载造成的。对这种情况解决的办法是：重装一遍应用程序，然后利用安装完成后的Uninstall选项进行卸载。(陈军)
　　江苏读者刘清丽问：我的电脑最近装了Windows 95，启动Windows 95后，选择“重新启动并切换到MS－DOS方式下”项，用CCED制表，打印时显示“内存不足”的提示。而在启动时，直接进入DOS却没有这种情况发生，请问从Windows 95状态启动切换到MS－DOS方式时，为何在CCED状态下打印时，会出现“内存不足”的提示?
　　答：估计你所说的“启动时直接进入DOS”是进入原来的DOS6.22版本(启动Windows时按下F4键实现)，而“重新启动并切换到MS－DOS方式下”所进入的DOS版本是DOS 7.0。CCED程序是在Config.sys和Autoexec.bat文件所设置的环境下工作的。DOS 6.22和DOS 7.0这两个版本共存在硬盘之中，而这两个不同版本的DOS使用的Config.sys和Autoexec.bat是不同的文件，它们的内容也可能不同，因此设置的环境也可能不同，这样就可能出现来信所说的现象。在启动Windows95(或者DOS7.0)时，DOS 6.22的Config.sys和Autoexec.bat文件将被更名为Config.dos和Autoexec.dos。因此建议你在DOS7.0下面，按照Config.dos和Autoexec.dos的内容修改Config.sys和Autoexec.bat文件，这样就可以避免来信所说的问题了。(何宗琦)
　　江西读者顾磊问：我的机器配置为：则灵586VIA－VPX(512K)主板，火球五代21G硬盘，分为C和D两个区，16M EDO内存，CPU为Cyrix 6x86MX－PR200，操作系统为Windows 95 OSR2。其间因为误删了System.da0而导致系统崩溃，后从其它电脑上拷贝了一份，系统又重新恢复，但在检查设备管理时，发现有的设备选项前出现惊叹号，包括：标准PS/2端口鼠标、PCI ISA Bridge、PCI Universal Serial Bus、Primary IDE Controller(single fifo)、Secondary IDE Controller(single fifo)，后两者的父设备为“Standard Dual PCI IDE Co
ntroller”，而系统性能一项中显示C、D盘正以MS－DOS兼容方式运行，降低了硬盘性能。我检查了Config.sys文件，除光驱驱动程序外，没有安装其它硬件设备驱动程序。请问如何才能使这些带惊叹号的项目恢复正常，特别是如何恢复硬盘原有的性能?
　　答：首先指出，你在信中说误删的是system.da0，但该文件只是Windows 95系统注册表的备份文件，删除后不会引起系统崩溃，故怀疑你实际删除的是System.dat。由于不同机器的硬件设备和控制芯片往往有很大差别，你机器上的Windows 95使用其它机器的配置当然会引起冲突。解决方法如下：如果你没有使用PS/2鼠标，那么删除该项；删除“PCI ISA Bridge”项，重启系统后，如果Windows 95又自动找到该设备，这时需要安装随主板带的驱动程序，如果没有找到，说明你的主板没有该设备或者已经安装了驱动程序；USB驱动程序的安装需要用到主板所配驱动盘或者Windows 95 OSR2的补丁程序(《电脑报配套光盘》上有)；最后两个惊叹号就是导致C、D盘以MS－DOS兼容方式运行的原因，由于不能直接删除它们，所以解决起来要麻烦一些，一种方法是在系统属性中复制一份硬件配置文件，把新配置文件中的Standard Dual PCI ID Controller删掉，重新启动机器，选择新配置文件并启动Windows 95，然后运行添加硬件向导，让它自动查找并安装IDE控制器，如果安装好后问题排除了，便可删掉老的硬件配置文件。如果问题依旧，则要使用第二种方法，即重装Windows95 OSR2，可就在Windows 95下运行Setup程序，并选择安装到原来的Windows目录下。考虑到你的特殊情况，笔者倾向于建议你直接使用第二种方法试试，顺便还可排除注册表中可能还存在的其它问题。(王琰)
