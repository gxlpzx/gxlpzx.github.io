WIN98-注册表使用技巧十六例
　　一、关闭WINDOWS 的动画以加快运行速度

　　找到HKEY_CURRENT_USER\ Control Panel\ desktop\ WindowMetrics，选择“新建”、“添加”、“字 符串值”，加入MinAnimate：0 (关）或 1 (开)。

　　二、缩短长文件名

　　选择HKEY_LOCAN_MACHINE\ System\ CurrentControlSet\ control\ FileSystem， 加入新的二进制值 NameNumericTail，设为 0 。这样，符号~ 和长文件名就不再显示了，只是如果有一个以上的和它的短文件名相同的文件，后面会加上数字。

　　三、不从“开始”菜单启动应用程序

　　你可以为“我的电脑”设置一个弹出菜单（按右键弹出），找到上面的关键字HKEY_LOCAN_MACHINE\ System\ CurrentControlSet\ control\ FileSystem，打开子键shell，建立一个新的子键名字类似于 contr1，其值设为&Control Panel，打开它， 建立子键COMMAND，修改它的默认值为 %windir%/control.exe，%windir%/explorer.exe c:/ 会打开C盘的根目录，加入参数/e，用这种方式加入你喜欢的程序。

　　四、修改注册表后无须重新启动

　　按 Ctrl+Alt+Del, 选择Explorer，按“结束任务”。当Windows 询问“是否关机”，选择“不”，然后再选择“结束任务”。

　　五、从桌面上移走系统图标

　　试着找到类似“收件箱”的东西，从它的子文件夹SHELLNEW中删除ATTRIBUTE。

　　六、从快捷方式中移走讨厌的小箭头

　　Windows95在所有快捷方式的左下角都加入了小箭头，你可以这样移走它们：

　　选择 HKEY_CLASSES_ROOT\.lnk ( Windows 程序的快捷方式)或者选择 HKEY_CLASSES_ROOT\.pif ( DOS 程序的快捷方式)，左击右边窗口的“IsShortcut”，按 DELETE 键。重新启动WINDOWS即可。

　　七、加速“开始”菜单

　　找到HKEY_CURRENT_USER\ Control Panel\ desktop ，在“desktop”上右击，选择“新建”和“串 值”，输入MenuShowDelay，并设它的值为1到10（1的速度最快！），重新启动WINDOWS即可！

　　八、使用三键鼠标

　　你可能希望将双击功能定在中间的键上，这种操作在控制面板中无法设定，可找到 \HKEY_LOCAL_MACHINE \SOFTWARE \Logitech\ MouseWare\CurrentVersion \SerialV \0000 ，设为 DoubleClick to 001。

　　九、修改某些重要的文件夹的名字和位置

　　找到注册表中的 \HKEY_CURRENT_USER \Software \Microsoft \Windows \CurrentVersion \Explorer \Shell Folders ，输入你要修改的值即可。

　　十、打开另一个窗口的文件夹 （在拷贝文件时很有用）

　　找到HKEY_LOCAL_MACHINE\ SOFTWARE\ Classes\ Directory\ shell，建立一个新的主键openw，设置它 的默认值为“Open New Window”（或者其它），加入新的子键command，默认值为explorer.exe %1 。

　　十一、从“控制面板”的“添加/删除”中删除应用程序

　　有时你会很奇怪，为什么在“控制面板”的“添加/删除”中会看到一些早已被删除的程序，但是执行时 又会报告“没有找到”！ 启动REGEDIT，找到Hkey_Local_Machine\Software \Microsoft\Windows\CurrentVersion\Uninstall，点击你想删除的文件名选择“删除”。

　　十二、改变应用程序的路径

　　启动REGEDIT，找到Hkey_Local_Machine\Software \Microsoft\Windows\CurrentVersion\AppPaths，找到你要修改路径的程序名，双击改变路径即可。

　　十三、从“开始”菜单中删除程序

　　选择Hkey_Local_Machine\Software\Microsoft\Windows\CurrentVersion\Run，找到要删除的程序名选 择“删除”。

　　十四、从运行菜单中删除路径

　　选择HKEY_CURRENT_USER\Software\Microsoft\CurrentVersion\Explorer\RunMRU\ ，删除除默认值以外 的所有路径。

　　十五、给应用程序加声音

　　通过WIN 95中的“控制面板”中的“声音”你可以发现一些预先设置好的声音事件，一些应用程序在安 装时也加入了声音，你可以用下面的方法给你喜欢的应用程序加入声音： 选择 HKEY_CURRENT_USER\AppEvents\Schemes\ Apps；右击 “Apps”，选择“新建” 、“主键”；输入要加入声音的程序名；右击刚才建立的主键，选择 “新建”和“主键”；输入要加入的事件名： ⊙Close ⊙Open ⊙Maximize ⊙Minimize 重新启动后即可。

　　十六、在WIN 95中有一些你不能删除的图标

　　如收件箱、网上邻居等，下面的方法可以帮助你：

　　找到 “HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\explorer\Desktop\NameSpace”，它 下面就是一些可以删掉的图标中值，删掉你讨厌的图标，重新启动吧!

Windows 98技巧手册


　　Windows 98简体中文版已成为时下的主流操作系统。与Windows 95相比，新版的操作界面变化并不算太大。因此，Windows 95的绝大多数使用技巧都适用于Windows 98。不过，作为操作系统的升级版，新特性中总是蕴藏着新的使用技巧。基于此，本文将Windows 98独有的使用技巧整理出来，力求全面和准确，以飨读者。

　　●把硬盘当光驱

　　很多人都知道，把安装文件拷贝到硬盘上安装，速度可以大大提高。其实，这样做还有两个好处：

　　首先，今后添加新硬件，或是对Windows的组件进行增删时，免去了重新插入光盘的麻烦。其次，如果首次安装不成功，在原启动软盘没装CD－ROM驱动程序的情况下，可直接从硬盘运行安装程序。Windows的安装程序压缩包不足100MB，对于目前动则几GB的硬盘而言，实在微不足道，以如此小的代价换来如此多的方便，何乐而不为呢？而复制的方法也很简单，直接将Windows 98光盘上的Win 98文件夹拖到硬盘即可。

　　●选择正确的FAT格式

　　Windows 98安装完成后，首先要考虑的就是使用何种FAT格式。FAT32是Windows 98最大的卖点，它所能管理的磁盘容量比旧的DOS和Windows文件系统多得多，而且保存文件的空间效率也高得多。因此，对于多数用户FAT 32是最佳的选择。但Windows 98的缺省安装仍使用FAT 16，如果要用FAT 32，必须使用Windows98附件组中提供的FAT格式转换器进行转换。不过，有以下情况的用户使用FAT 32并不能带来多少好处。

　　1、硬盘的容量小于1GB。

　　2、当前硬盘已使用DriveSpace或其它种类的磁盘压缩程序，这是由于FAT 32不兼容此类程序。

　　3、具有“冬眠”工作模式的旧式笔记本电脑，尤其是早期的Phoenix BIOS无法将内存中的信息保存到FAT32格式硬盘。（在“冬眠”模式下，BIOS可将内存中的内容以文件的方式保存到硬盘，下次开机时再注入内存以恢复关机前的状态。）

　　4、原有系统正在使用非为Windows 98开发的磁盘管理软件，比如旧版的Norton Utilities、 Partition Magic、Ontrack Systems Disk Manager、V Communications System Commander、Iomega Jaz tools和SyQuest设备驱动程序等。这些程序都与FAT 32不兼容。

　　如果你不属于上面任何一种情况，就应当使用FAT 32。另一方面，对于原来就使用Windows 95 OSR2版的用户，很可能原来就已在使用FAT 32了。如何判断当前系统是何种格式呢？双击“我的电脑”图标，打开驱动器盘符的属性，如果看到“文件系统：FAT32”的字样，表明系统已是FAT 32了。

　　用户界面的定制

　　如果你是Windows 95的老手，就会觉得Windows 98的界面与Windows 95十分相似，但用上一段时间，也会发现新版本界面上的变化体现在许多细微之处。这些小变化，未必处处合你心意，不过，你完全有理由按自己喜欢的原有方式行事。

　　●关掉动画显示

　　第一次启动Windows 98，你首先发现打开菜单、移动文件等操作时都伴随着活泼的动画显示。不过，这些动画会占用更多的系统资源，而且降低了操作的反应速度，相信多数人很快就会厌倦。你可用两种方式来关闭这种小孩子玩的把戏。

　　方式1：右击桌面，选择“属性”，指向“效果”选项卡。在“视觉效果”组中，关闭“动画显示菜单”和“拖动窗口时显示其内容”项即可。

　　方式2：使用Windows 98光盘附送的界面调整程序TweakUI。该程序不会随Windows 98安装到硬盘，用户可另行安装。在安装了该程序后，可双击“控制面板”中的TweakUI图标，选中“常规”选项卡，关闭有关动画显示的项目。（注：TweakUI位于Windows 98光盘的“Tools/ResKit/Powertoy”文件夹下，只需右击 tweakui.inf，选择下拉菜单中的“安装”即可。）

　　●有选择地使用资源管理器的左窗口。

　　传统的资源管理器左窗口显示的是当前右窗口文件所在的路径。Windows 98提供了更多与网络相关的显示方式，通过“查看”菜单中的“浏览器栏”，你可以让资源管理器的左窗口以另外4种方式显示：搜索、收藏夹、历史和频道。这些方式有助于你迅速进入网络世界。

　　●给任务栏增加项目

　　Windows 98的任务栏分为三个区，最右边是系统区，中间是活动任务区，靠近“开始”按钮的是快捷方式区。在缺省方式下，快捷方式区配置的是“快速启动”工具栏，它包含了常用的程序图标，可以快速通过单击打开。不过，你完全可以将自己常用程序的快捷图标加入到这个快速启动区，无论是文件夹、文件还是快捷方式，只需将其从桌面上拖动这个区就行了。反过来，任务栏上的图标也可随意拖动到桌面和“开始”菜单中。

　　●创建个人工具栏

　　除了向快速启动栏添加新项目外，你还可以独立创建个人专用的工具栏，这在几个人共用一台PC时最为有用。创建步骤：将所有喜欢用的快捷方式拖到一个新的文件夹中，右击任务栏上的空白区，从菜单中选择“工具栏”，选择“新建工具栏”，在浏览对话框中选中刚才建立的文件夹，再按“确定”即可。 新的工具栏出现后，可能带有文本标签，占用了太多空间，可在新工具栏上右击，选择“显示文字”可关闭。原快速启动栏中的快速切换到桌面图标是一个相当实用的快捷方式，在创建自己的工具栏时最好别放弃。

　　●改变任务栏的图标

　　如果你同时将两个同类型的快捷方式拖到任务栏上，可能会由于其采用的图标相同而无法加以区分，这时可考虑给它们以不同的图标。右击要改变项，选择“属性”，按下“更改图标”，选取任意一个，或者按下“更多…”，从不同的文件中抽出图标。如果你有IconEdit Pro或TurboIcon这样的图标编辑工具，还可创建个人特色的专用图标。

　　●清洁桌面

　　当你使用了活动桌面，并在桌面上添加了许多活动项目后，桌面上原有的“我的电脑”、回收站、网上邻居等图标就成了视觉污染。不过，我们有办法让桌面变得清洁起来。最简单的办法是右击桌面，打开“属性”，选中“效果”选项卡，使“当桌面以Web方式显示时隐藏图标”项有效。此后，你只要以Web方式查看桌面时，这些图标都是不可见的，但以传统方式查看时，就会恢复可见。

　　即使是在传统桌面方式下，有些图标你可能也不喜欢，比如“MSN网络登录”、“我的公文包”等，最简单的方法是用右键菜单中的“删除”命令删除。这样做虽然省事，但万一将来要使用，恢复起来就很麻烦。比较保险的做法是使用TweakUI，指定哪些项目可以在桌面上显示，哪些不能。特别地，一些特殊的图标，如“控制面板”和“回收站”是可以用TweakUI保存为文件，并放置到“开始”菜单中的，这方便了一些

　　常用程序的调用。

　　●为文件夹设置背景

　　如果你不希望每个文件夹都千篇一律，可以考虑给不同的文件夹以不同的背景，以便于迅速识别，这特别适用于设置一些特殊的文件夹。你还可以将自定的GIF图用作背景。步骤是，单击“查看/自定义文件夹”可启动自定义向导，之后选择“选择背景图片”，即可在列表框中选择当前目录下的BMP和GIF文件作为背景。你也可使用预览来预看效果。

　　文件管理与操作

　　Windows 98的改变不仅是在界面，它的整个运行环境也有所变化。对这些变化的掌握可提高操作效率，给人以流畅感。下面这些小技巧也许不引人注意，但对于一个熟练的PC用户而言，却是必须掌握的。

　　●活用收藏夹

　　Windows 98的“开始”菜单新增了一个收藏夹的选项，其中主要是IE收藏的网址列表。不过，能放进收藏夹的不仅仅是网址列表。启动资源管理器，打开文件夹C:\Windows\Favorites\，你可以把常用的程序、文件夹及文件的快捷方式统统放进去，以后你就可随时从这里启动常用的程序，打开常用的文档了。

　　●快速打开和关闭活动窗口

　　当活动桌面上的项目很多时，可能彼此间会妨碍工作，如果一个个地关闭，实在麻烦。 有两种方法可实现快速打开和关闭活动窗口。即，单击任务栏上快速启动工具栏的桌面显示按钮，或者按下微软键盘的Windows＋M键。反之，若要恢复所有的窗口，可按Shift＋桌面显示按钮，或Shift＋Window＋M键。

　　●快速获得帮助

　　当你想查找帮助文件时，通常是使用“开始”菜单中的“帮助”，但这样系统会自动拔号并连接到微软的帮助站点，不胜其烦。有一个相当直接的方法，就是打开Windows\Help文件夹，为文件windows.hlp在桌面上创建一个快捷方式，以后直接调用。步骤：选中windows.hlp，按住右键拖动到桌面，释放按钮，在弹出式菜单中选择“创建快捷方式”。

　　●让文件扩展名和隐藏文件现身

　　在缺省设置下，Windows 98的文件夹关闭了扩展名的显示，这使得文件类型的判断十分不便，通常我们最好还是将它显示出来。选择“查看”下的“文件夹”，转到“查看”选项卡，关闭“隐藏已登记文件扩展名”项。这样，你才可能区别出tweakui.inf、tweakui.hlp和tweakui.cnt这三个文件有何不同。

　　同样，在缺省设置下，Windows 98不会显示具有“隐藏”属性的文件。这常常造成这样的麻烦，你明明看见一张软盘上没有文件，但可用的空间却只有几百个K，你不会想到是软盘上的隐藏文件在作怪。选择“查看”下的“文件夹”， 转到“查看”选项卡，打开“显示所有文件”项。但如果你的电脑常有新手使用，则还是不显示隐藏文件为好，以防它人误删除系统文件。

　　●文件列表项目重排

　　在资源管理器窗口中，以详细列表方式显示时，可能缺省的顺序不合你用。此时，直接用左键拖动至新位置，就可完成重排。比如，你可将日期拖到文件类型的前面。至于要调节各栏的宽度，将光标移到两栏交界处，当光标变成双箭头时，左右移动即可。在缺省状态下，系统按名字顺序排序，但你只要单击任一栏名，就可按此栏排序，再击则是反序。

　　●禁止系统启动时运行某程序

　　一些应用软件在安装时，会设置为开机就启动。而对于一些不常用的软件，开机即启动是令人讨厌的。如果该程序是通过“开始”菜单中的“启动”文件夹实现自动加载的，可将Windows目录下的“启动”文件夹中的快捷方式清除即可。但一些应用程序是通过修改System.ini和Win.ini来自动加载，控制起来就麻烦很多。不过，Windows 98自带了一个系统配置实用程序，可以快速禁止自动运行程序的加载。步骤：指向“开始/程序/附件/系统工具/系统信息”，选择下拉菜单“工具”下的系统配置程序。启动程序后，按下“常规”卡中的“备份”按钮，备份当前的系统配置。转到“启动”选项卡，此时你可看到许多程序的列表，如果你不希望加载它们，只需单击复选框，消除选中符号即可。实际上，“启动”选项卡中列出的程序包括了“启动”文件夹、System.ini和Win.ini 三个地方的自动加载项设置。

　　●为活动桌面添加新的Web项目

　　Windows 98的活动桌面允许你将各种Web项目添加到桌面上，这些项目可以是基于Java的，也可以是含有ActiveX控件的，还可以是动态HTML乃至Navigator的书签文件。右击桌面上的空白处，选择弹出式菜单中的“活动桌面/自定义桌面”，转到“Web”选项卡，确保“按Web页方式查看活动桌面”项处于开启状态。 再单击“新建”按钮，如果你想使用微软提供的动态气象图、Java时钟和一些新鲜玩艺儿，就在一个对话框询问你是否访问微软画廊时选择“是”，如果你想用自己的材料，选择“否”。下一步，选择“浏览”，就可将你喜欢的Web项目放到桌面上，这些项目将以列表的形式出现在显示属性对话框中，最后按确定即可。

　　●把墙纸作为记事板

　　在大多数情况下，你不必把花里胡哨的Web项目放在桌面上，相反，把易于编辑的文本信息放到桌面上提醒每日要做的工作则更为实用。现在来看看如何在桌面上创建记事板：打开笔记本，用HTML语言编辑你要加入的信息内容，如果你想省事，只写文字部分也可，再以HTML扩展名保存该文件。右击桌面空白处，选择“活动桌面/自定义我的桌面”，在“Web”选项卡上，使“按Web页方式查看活动频道”有效，单击“新

　　建”按钮，有对话框询问是否到微软画廊时，选择“否”，接下来通过“浏览”将刚才创建的HTML文档作为Web项目添加到桌面上。返回桌面，可以见到刚才创建的文档已在桌面上显示。

　　系统维护

　　Windows 98作为一庞大的操作系统，是需要我们经常花点时间进行维护的。如果你长期懒于系统维护，用不了多久，Windows 98就可能慢如蜗牛，或者干脆在你工作最忙的时候DOWM机。

　　●保持系统处于最新

　　要想与时代同步，电脑系统的及时升级是必不可少的。就Windows而言，微软几乎总是在推出新的补丁程序和服务包，而这些小小的升级通常可使系统运行更加稳定和快速。因此，定期使用Windows 98基于Web的升级向导是一个好习惯。在操作上则是很轻松的，选择“开始/Windows更新”系统就可自动连接到微软的升级站点，此后按照提示做下去即可。

　　●果断返回升级前的状态

　　在强调及时升级的同时，也应根据实际情况果断放弃不成功的升级。一些升级组件不是人人都适用的，遇到此种情况，及时走回头路同样是明智之举。有幸的是，微软的升级站点会记录整个升级过程，如果你想后悔，只要及时选择卸载，就可返回原状态。

　　●把常用工具放在手边

　　Windows 98提供了许多功能强大的系统维护工具，不过，这些工具都深藏于重重菜单之中，如磁盘清理、系统信息、注册表检查器。把这些常用工具放在顺手的地方，是提高系统维护效率的诀窍。好在现在Windows 98支持菜单项的拖放操作，你在“开始”菜单中看中了某项，可用鼠标直接将其拖至桌面上或任务栏上。笔者建议为常用的工具创建一个单独的工具栏。

　　●定期检测系统的完整性

　　Windows 98是一架笨重而又精密的机器，它要求应用程序、驱动程序、动态链接库相互间紧密配合，不能有一丝差错。因此，系统日常维护工作的一个重要内容就是定期检查系统文件的完整性，而系统文件检查器则是一个得力工具。选择“开始/程序/附件/系统工具/系统信息”，再从“工具”菜单中选择“系统文件检查器”，选中“改变过的文件”，系统将测试所有的程序文件及OCX、DLL等驱动程序和关键系统文

　　件。如果发现错误，则会提示你插入光盘进行恢复安装。

　　●使用压缩代理

　　如果当前的硬盘是一个压缩盘，可能会有两件事让你头痛：首先，在系统实际空间不足以安装某软件的情况下，系统却报告足够。其次，压缩数据的读取增加了CPU的处理时间，速度上也就打了折扣。因此，最好能在空间利用和速度两者之间获得一个相对平衡的选择。Windows 98的压缩代理试图给你这种平衡，它能将不常用的文件以高压缩率保存，而对于常用的文件以低压缩率保存，以提高读取速度。启动压缩代理的位址是“开始/程序/附件/系统工具/压缩代理”。

　　●从CAB包中抽取文件

　　所有Windows 98安装源文件均以压缩方式保存在光盘上的CAB压缩包中，通常无法直接调用某个文件。不过，你可以用Windows 98的系统文件检查器来抽取需要的一个或几个文件，其步骤是：通过“开始/程序/附件/系统工具”启动系统文件检查器，并使“从安装盘中提取文件”项有效，并在空白栏中键入文件名（不允许通配符），再单击“开始”。在接下来的对话框中，通过“浏览”功能向系统指定Windows 98安装

　　光盘所在的路径。找到所要的文件后，再指定保存路径即可。

　　●修复FAT 32对剩余空间的报告错误

　　FAT32有一个缺陷，就是它计算磁盘的剩余空间不太准确。有多起报告说，当磁盘还有200M空间时，FAT32文件系统却报告磁盘已满，这通常发生在系统崩溃以后。解决的办法是运行ScanDisk：在“我的电脑”或资源管理器窗口中，右击驱动器盘符，打开“属性”，转到工具选项卡，在“查错状态”组中按下“现在检查”按钮即可。

　　如何实现Windows 98 和Windows NT 4.0 的共存

　　Windows 98 和Windows NT 4.0 可以共存，并且可以方便地切换，其方法是：先在机器上安装Windows 98，然后安装Windows NT 4.0，如果想让Windows 98 能够访问Windows NT 4.0 的文件，那么Windows NT 4.0 就只能安装于FAT 文件格式的硬盘上（因为Windows NT 本来的文件分配格式是NTFS 的），这会失掉一部分Windows NT 4.0 的安全特性。

　　在启动或者运行Windows 98时的VxD错误

　　●:EXTRACT [/Y] [/A] [/D] [/E] [L DIR] CABINET [FILENAME]

　　●:EXTRACT [/Y] SOURE [NEWNAME]

　　●:EXTRACT [/Y] [/C]SOURE DESTIANATION

　　CABINET指定Windows 98安装盘中的:CAB文件(含有两个以上的文件)，

　　FILENAME指定 想从CAB文件中解压缩的文件名,可以使用通配符或多个文件名(以空格隔开 )

　　SOURE包含一个文件的CAB文件

　　NEWNAME为:解压缩后的文件名,缺省则用原文件名

　　/Y:覆盖已存在文件时不再提示

　　/A:处理全部CAB文件,从指定的第一个CAB开始

　　/D:显示CAB文件中所包含文件的目录,不进行解压缩

　　/E:解压缩,取代*.*,即解压缩所有的文件

　　/L dir:指定解压缩文件的位置,缺省为当前目录

　　/C:拷贝CAB文件

　　例如:从WIN98_27.CAB文件中解压缩cdfs.vxd文件:，可执行如下命令即可，而不需要 重新安装Windwos 98：

　　EXTRACT/Y WIN98_27.CAB cdfs.vxd

揭开ATI Radeon 256的面纱


　　现在的硬件发烧友越来越追求最新的硬件技术。而 ATI Radeon 256 一直是他们心中所期待的一块显卡，为了迎接这块显卡的到来，他们已经等了很久。今天，他们终于等到了，对于游戏发烧友更是一个令人感到兴奋的日子。

　　ATI 终于发布了这款名为 Radeon 256 的新一代显卡，在此之前， ATI 曾把它命名为 RAGE 6 。为了研制这块显卡 ,ATI 公司花了很长时间进行产品的研发，他们的最终目标是取得高端显卡和游戏市场的龙头地位。从 ATI 公司的决心也能看出，这绝对是一块非同一般的显卡。

　　Radeon 256 的主要性能指标：

　　作为市场上倍受赞誉的第二块 GPU ， Radeon 256 并没有按照 ATI 公司的传统来命名。 -Charisma Engine- 这项技术包括了对最先进的硬件几何变形，光照效果，和图象剪切的完全支持。并且 Radeon 的像素毯体系结构（ -Pixel Tapestry Architecture- ）能够提供更为先进的多纹理支持并通过混合处理过程来提供更为真实的影院效果。

　　ATI 采用硬件 T & L 技术无论对于消费者，研发人员， ATI 公司甚至是 NVIDIA, 都是一个好消息。即使是 T & L 的开创者 NVIDIA 也只是采用了 S3 公司的一些技术但他们的驱动程序做的并不理想，以至实际效果差强人意 ，而 ATI 公司在这方面一直是强项。从理论上讲，要想在硅材料上开发出新的特性已经相当困难了。所以只有利用现有的方法将性能提高，即对系统进行进一步的优化使之发挥出更高的性能。而这对于研发人员来说将是一个极大的挑战。 ATI 公司一直在努力开发新的技术并将其运用到最新的产品中。这也是 ATI 公司研发实力的表现。

　　在设计 Radeon 256 的过程中，另一个复杂且困难的工作是对于通过硬件加快关键帧的插入， 3D 纹理的支持，及对三种贴图的支持 ---Direct-X Bump 贴图（ Direct-X Bump Mapping ） ，环境贴图（ Environment Mapping ） 和阴影贴图（ Shadow Mapping ）。

　　Radeon 256 同样提供硬件级的顶点皮肤渲染，笔者认为这是个非常先进的技术，至少比 NVIDIA 提供的要强，如果能够得到软件开发者们的支持及硬件厂商的肯定，那它一定会成为一个新标准，从而使软件和硬件的配合更紧密。具有四个建模的 Radeon 256 比起 NVIDIA 的两个建模要精确的多，并且速度也相当快，当然速度是很重要的一个方面。在我们没有对这两块显卡进行具体的实际评测之前，很难说到底是 NVIDIA 的 2x 快，是 ATI's 4x 的快。

　　在 Radeon 256 所有的新特性中，顶点皮肤渲染被证明是最具有实际利用价值的技术因为它能利用现有的格式构造出所需的三维模型，这一点在 3DStudio Max 中得到了证实。如果有更多的软件开发者和硬件制造商来支持的话，相信前景将非常广阔。

　　关键帧插入（ key frame interpolation ） ------ 象 FLASH 中那样，你只需定义一个物体的几个关键布局点的帧，其他的过渡帧都由硬件来自动生成。这就可以节省大量的内存空间和带宽。但这项新技术不一定马上能够被其他的硬件厂商所接受，关键的原因在于研发人员没有足够的资金来使这项新技术运用到不同的硬件上并保持兼容性。

　　另一方面， ATI 已经通过合并一家德国芯片公司来完善他们的 RAGE “影院芯片”（ Theatre companion ） 并以此来占领显卡市场。

　　Radeon 256 能够统治图形加速卡的市场吗？下面我们来看一下这块芯片的实际性能，你就会有自己的答案了。

　　通过跳跃性的使用 0.18 微米的芯片制造技术， ATI 能够在他们的第一代具有 T&L 硅原料的芯片上集成更多的晶体管。事实上， Radeon 256 在芯片上集成了超过两百万个晶体管，甚至超过了 Intel Pentium III 1GHz CPU ，这使它成为目前最先进的 GPU 。尽管已经是很高的集成度，但 ATI 还是设法提高了时钟频率。

　　众所周知， ATI RAGE 128 PRO 主要是依据 RAGE 128 来设计的，他们在整体的构架上几乎一模一样。而这次的 Radeon 256 不同， ATI 公司重新设计了芯片，采用了完全不同的构架，目的是要取得历史 性破。 200MHz 的双像素通道，每个通道有 3 个纹理单元，那么最大的纹理填充率就等于：200MHz*2 个通道 *3 个纹理单元 =1200M 像素 / 秒

　　有必要指出，填充率只有在三个纹理用来表示像素后才能被实现。

　　也许你对 Z-BUFFER 这种新技术感兴趣，它能同时提高填充率和带宽。我们得知 ATI 的专利技术能够减少显卡 Z-BUFFER 对带宽的利用率。结果是在一个游戏场景中，填充率一下子从 1.2G 像素 / 秒跳到了 1.5G 像素 / 秒，而带宽从 6.4GB/ 秒跳到了 8GB/ 秒。在我们 没有拿到这块显卡之前我们不能肯定这种技术所带来的性能提升是否会引起负面效应。

　　显存：

　　随着游戏越来越复杂，对像素填充率的要求越来越高，更大的帧缓冲也需要更多的显存来存放更多的纹理单元。然而与流行的观念相反，内存的容量并不总是最重要的因素。为了尽可能快的移动显存中的数据，带宽的大小同样很重要（特别是在 32 位色深高分辨率下）。

　　依靠 MAXX 技术， ATI 已经使他们在提供内存带宽的技术方面取得领先地位，使得 ATI 的产品能够在 32 位色深高分辨率下提供很好的性能。 Radeon 256 不只仅仅是超越这项技术，而是从头开始采用单芯片来提供更大的内存带宽。

　　ATI 聪明的选择了同时支持 SDR 和 DDR 内存，容量可高达 128M 。由于最高有 200MHz 的频率，这块显卡的理论带宽可达 8GB/S 。这些带宽能被全部用上吗？或许不能，因为就连当今装备了 DDR 显存的显卡也只能在 1280x1024 下用上全部带宽，但作为 Charisma 引擎的特性之一，在高色深高分辨率下提供足够的带宽是必要的。

　　上个月，我们深入研究了 Radeon 256 的一些新的技术特性。下面我们将会提供一个比较完整的报告。

　　让我们再次关注一下“ Charisma Engine ”

　　ATI 参与设计 Charisma 和 Pixel Tapestry 引擎主要是为了能够在屏幕上再现栩栩如生的真实图象。而实现这些影像， ATI 主要通过硬件级的 T&L, 图象剪切，顶点皮肤渲染，和关键帧补偿。如果显卡能够在一秒中内处理 30M 个多边形的话，那么游戏开发者就能够用更多的多边形来构造物体，从而使场景中的物体更为真实。

　　ATI 花了相当大的精力来研究硬件光照算法，为了保证在最多使用八个光源的时候系统的性能损失减到最小。以下的图片用来对比标准的 T&L 引擎（ GeForce 256). 当使用八个光源的时候差不多有 60% 的 ATI 引擎的几何处理能力可以用来进行图象变换功能。虽然我们的实验室并没有证实这些数据，但在最近的一次关于 ATI 产品的会议上，我们就已经看到这个产品的性能了。

　　插入关键帧

　　利用现在的 3D 图形卡来产生复杂的动态的人物，需要占用大量的带宽和显存。举例来说，要实现人物的脸部表情的变化需要游戏美工用某些动画软件包来产生大量的过渡帧。当创建一个连续的动作时每一帧都必须作为一个单独的模型储存在游戏中，并连续的进行播放。正因如此，我们很少在游戏中见到真实的面部表情。而如果利用 ATI 的关键帧补偿来实现的话，就有可能使用硬件来完成关键帧之间的过渡，我们所要做的仅仅是给出初始帧，中间帧，结束帧，其他的都通过硬件来完成。尽管实现这一特性需要软件游戏开发者们的支持，但是 ATI 公司一直坚信他们能够赢得足够多的支持来使得这一技术得以普及。让我们拭目以待！

　　顶点皮肤渲染（Vertex Skinning）

　　还有一种方法能够使的人物模型更加鲜活，那就是骨骼动画，它能使人物模型的身体随着由一系列顶点构成的骨骼而伸展。当骨骼移动的时候，顶点也会相应的重排。

　　但是，这种动画方法有一个弱点，那就是在它们的触点衔接处。当一个变换导致关节弯曲的时候，就有可能产生裂缝，从而使人物模型看上去很不自然。这个弱点可以通过“皮肤渲染”的技术来克服。它可以调整顶点，来产生一个连续的弯曲的关节，并对其进行修复，使物体更加真实。

　　为了产生皮肤渲染效果，通过变换顶点坐标来计算顶点位置。为了使每一块骨头有一个给定的顶点，必须有一个附加坐标来加强皮肤渲染。使用 Charisma 引擎，最多可以使用硬件进行四个建模的计算。而 GeForce 只支持两个。同样的，要使这一硬件特性成为一个优势，离不开软件开发者的支持。

　　像素毯（Pixel Tapestry）：

　　Pixel Tapestry 是 Radeon 256 的非常重要的一个特性。也是一个非常先进的技术。下面我们来谈一下关于“像素毯”这项技术。最重要的是像素毯提供了对三纹理通道的支持，并应用到了每一个像素的传送中。它的一个重要的应用是软件开发者能够在未来的应用程序中加入这一特性来提高软件（游戏）的真实性。

　　最重要的是，增加的纹理单元可以使场景中产生更多的细节，使影像质量更高，而内存带宽并不受影响。下面的图片反映了各个纹理通道及组合后的效果。

　　大多数给人留下深刻印象的影像效果都是在多边形的表面上进行纹理贴图来产生的。当用软件实现这些效果的时候，并不能充分的表现出全部的效果，因为这会大大的加重 CPU 的负担。为了解决这个问题， ATI 开发出利用硬件进行纹理转换的“像素毯”技术。 Radeon 256 能够计算出球面环境贴图，双抛物面贴图和立方体环境贴图。

　　碰撞贴图（ Bump Mapping ）

　　正如我们先前所述， Radeon 256 是世界上第一块用硬件来实现三种碰撞贴图方法的芯片。每种技术从复杂程度和应用有效性方面来说，都有不同之处。所以需要有一种能够把这三种技术容于一身的新技术。我们还从没有看过有哪种游戏能够广泛使用碰撞贴图。从目前的情况来看，只有 ATI 对这些新技术提供了广泛的支持。

　　揭开内幕：

　　目前看来，最有价值的特性就是提高各种新技术的实用性。例如，全屏抗锯齿就是被 3dfx 大力推崇的一个特性。并且被许多小厂家所接受。同样， ATI 也站在潮流的前端。 ATI 最主要的目标是使用更高的分辨率和 32 位色深，而采用全屏抗锯齿的 TV-OUT 并不适合运行在高分辨率下。

　　和 3dfx 的方向一样， Radeon 256 将支持深场景和动态模糊效果，并提供能和 Voodoo 5 抗衡的产品。动态模糊的 DEMO 给我们留下了深刻的印象。

　　视频支持

　　ATI 的视频质量会更上一层楼吗？长久以来， ATI 的产品一直是公认的具有最好的影象质量和 DVD 解码特性，而这些特点在 Radeon 256 得以加强。为了保持 ATI 一向良好的名声， Radeon 256 提供了对硬件运动补偿 , iDCT, 和 alpha 混合的支持及增加了自适应的交错扫描方\式和一个带有完全 HDTV 解码的 165MHz 的 TMDS 的芯片。虽然最终的板卡还没有正式发行，但我们已得知 Radeon 256 将支持数字平板显示器接口，并且支持高达 1600x1200 的分辨率。

　　因为“自适应的交错扫描方式”（ adaptive deinterlacing ）在 PC 视频加速领域中还是一个非常新的技术。所以我们有必要对此进行一些解释。传统的显示器的工作原理是这样的，当一个视频流被交错扫描时，通过合并两个场来生成一个输出帧。而这过程中有大约 1/60 秒的延迟。如果在 1/60 秒中有动态的帧，那么在联合场中就会产生一种名为 motion artifact 的现象。

　　要避免 motion artifacts 这种现象的产生，可以通过调整图象的大小，使之尽可能的与真实的物体尺寸相符，还可以在通道间进行修改，以得到更为精细的图象。但随之带来的后果是会使图象变的不清晰，并要牺牲一部分图象的细节。

　　自适应的交错扫描方式（ adaptive deinterlacing ）就是基于这点而产生的。与调整图象大小的方式不同，自适应的交错扫描方式（ adaptive deinterlacing ）允许解码器对两个帧进行比较，从而决定在两个帧之间会产生什么样的动作。这种算法可以在图象运动的区域运用柔化效果，而在静止的部分中并不改变图象本来的质量。只要得到合适的软件支持，那么这种新技术将能有效的提高隔行交错扫描的质量。

　　很明显， ATI 投入了大量的精力用于开发 Radeon 256 芯片。在过去的两年里， ATI 动用了四个研究所用于开发这个项目。 ATI 这么做是否值得？我们认为这是应当受到肯定的。 ATI 曾经花了很大的力气来重新编写 Rage 128 的驱动程序的核心部分，现在这个驱动程序已相当完美，并完全为 Pentium III 做了优化，支持 SSE 指令集。 Radeon 256 的驱动程序的核心要想象 Rage 128 那样完全成熟还有很长的路要走，如果 ATI 确实能够象他们先前许诺的那样对驱动程序进行彻底的优化的话，那么我们可以肯定在高端的游戏市场上将有一场激烈的竞争。

　　随着 Radeon 256 的正式发行，许多我们先前的判断都将得到证实。但你有没有想过 ATI 会在这款产品中使用 MAXX ？实际上， Radeon 256 完全支持 ATI 的交替帧叠加技术（ Alternate Frame Rendering ）。也许未来的某一天我们将有机会看到使用双 Radeon 芯片的产品。你可以想象一下一块拥有 16GB/s 的显存带宽 , 3.0 GTexels/s 的像素填充率的双 Radeon 256 芯片的显卡会有什么样的惊人效果，当然也许你得掏空你口袋里所有的 MONEY ，才可能买的起这块显卡。

　　我们一直期待能有一个竞争者来面对 NV15 和 Voodoo5 。 ATI 已被 Q2 提名，从我们的角度来看，这块显卡好象是专门为 Q2 所设计的。 ATI 的热衷者们等待这块显卡的发布已有好几个月了，而 ATI 也意识到他们的错误，他们决定不再让零售商和顾客们苦苦等待了，所以他们决定立刻推出这块显卡。

　　ATI 的 2 通道/3 纹理单元技术在高端市场相当有竞争力，并且拥有极大的填充率，大的带宽，和一些非常先进的特性，这一切都让我们有理由相信， Radeon 256 正是我们一直期待的。
